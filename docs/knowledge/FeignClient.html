Feign声明式HTTP客户端

Feign是Netflix开源的声明式HTTP客户端，通过接口和注解的方式调用远程服务，让HTTP调用像调用本地方法一样简单，实现服务间的优雅通信。

❓

什么是Feign

Feign是声明式REST客户端，通过注解定义接口，自动生成实现类。类似于MyBatis的Mapper接口方式，让调用远程服务像调用本地方法一样简单。

flowchart LR A[服务消费者] -->|Feign调用| B[Feign Client] B -->|HTTP请求| C[服务提供者] C -->|返回数据| B B -->|返回对象| A

Feign vs RestTemplate

对比项RestTemplateFeign
代码风格命令式编程声明式编程
代码量较多（手动拼接URL）简洁（接口+注解）
可读性一般高（像调用本地方法）
维护性URL分散，难维护接口集中，易维护
负载均衡需手动集成Ribbon自动集成
服务降级需手动实现支持Fallback

基本使用

1. 启用Feign

Java
@EnableFeignClients
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

2. 定义Feign客户端

Java
@FeignClient(
    name = "user-service",           // 服务名
    url = "http://localhost:8081",   // 指定URL（可选）
    fallback = UserServiceFallback.class,  // 降级处理
    configuration = FeignConfig.class      // 自定义配置
)
public interface UserFeignClient {

    @GetMapping("/user/{id}")
    User getUser(@PathVariable("id") Long id);

    @PostMapping("/user")
    User createUser(@RequestBody User user);

    @GetMapping("/users")
    List<User> listUsers(@RequestParam("name") String name);

    @RequestMapping(method = RequestMethod.GET, value = "/search")
    User search(@RequestHeader("X-Token") String token,
                @RequestParam("keyword") String keyword);
}

3. 使用Feign客户端

Java
@Service
@RequiredArgsConstructor
public class OrderService {

    private final UserFeignClient userFeignClient;

    public Order createOrder(Long userId) {
        // 像调用本地方法一样调用远程服务
        User user = userFeignClient.getUser(userId);

        Order order = new Order();
        order.setUserId(userId);
        order.setUserName(user.getName());
        return order;
    }
}

配置详解

@FeignClient注解参数

参数说明示例
name/value服务名（注册中心）name = "user-service"
url直接指定URL（绕过注册中心）url = "http://localhost:8081"
fallback降级处理类fallback = UserFallback.class
fallbackFactory降级工厂（可获取异常）fallbackFactory = UserFallbackFactory.class
configuration自定义配置类configuration = FeignConfig.class
path统一路径前缀path = "/api"
contextId防止Bean名称冲突contextId = "userClient"

YAML配置

YAML
feign:
  client:
    config:
      # 全局默认配置
      default:
        connectTimeout: 5000      # 连接超时
        readTimeout: 5000         # 读取超时
        loggerLevel: full         # 日志级别

    response:
      enabled: true

服务降级

Fallback实现

Java
@Component
public class UserServiceFallback implements UserFeignClient {

    @Override
    public User getUser(Long id) {
        // 返回默认值
        User user = new User();
        user.setId(id);
        user.setName("默认用户");
        return user;
    }

    @Override
    public User createUser(User user) {
        throw new RuntimeException("用户服务不可用");
    }

    @Override
    public List<User> listUsers(String name) {
        return Collections.emptyList();
    }
}

FallbackFactory（可获取异常信息）

Java
@Component
public class UserServiceFallbackFactory implements FallbackFactory<UserFeignClient> {

    @Override
    public UserFeignClient create(Throwable cause) {
        log.error("用户服务调用失败", cause);
        return new UserFeignClient() {
            @Override
            public User getUser(Long id) {
                return User.builder()
                    .id(id)
                    .name("服务降级用户")
                    .build();
            }
            // ... 其他方法
        };
    }
}

// 使用FallbackFactory
@FeignClient(name = "user-service", fallbackFactory = UserServiceFallbackFactory.class)
public interface UserFeignClient { }

自定义Feign配置

自定义配置类

Java
@Configuration
public class FeignConfig {

    // 日志级别
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }

    // 超时配置
    @Bean
    public Request.Options options() {
        return new Request.Options(5000, TimeUnit.MILLISECONDS,
                                   10000, TimeUnit.MILLISECONDS, true);
    }

    // 重试策略
    @Bean
    public Retryer feignRetryer() {
        // 初始间隔100ms，最大间隔1s，最多重试3次
        return new Retryer.Default(100, TimeUnit.SECONDS.toMillis(1), 3);
    }

    // 请求拦截器
    @Bean
    public RequestInterceptor requestInterceptor() {
        return template -> {
            // 添加全局请求头
            template.header("X-Source", "feign");
            template.header("X-Request-Time",
                String.valueOf(System.currentTimeMillis()));
        };
    }
}

请求拦截器

Java
@Component
public class AuthFeignInterceptor implements RequestInterceptor {

    @Override
    public void apply(RequestTemplate template) {
        // 从上下文获取Token
        String token = SecurityContextHolder.getContext().getToken();

        // 添加请求头
        template.header("Authorization", "Bearer " + token);

        // 添加查询参数
        template.query("source", "feign");
    }
}

与Ribbon整合（负载均衡）

Feign默认集成Ribbon实现客户端负载均衡，通过服务名从注册中心获取实例列表：

graph LR A[Feign Client] -->|1. 获取服务列表| B[服务注册中心] B -->|返回实例列表| A A -->|2. 负载均衡选择| C[实例1] A -->|2. 负载均衡选择| D[实例2] A -->|2. 负载均衡选择| E[实例3]
YAML
# Ribbon负载均衡配置
user-service:  # 服务名
  ribbon:
    NFLoadBalancerRuleClassName: com.netflix.loadbalancer.RoundRobinRule
    ConnectTimeout: 1000
    ReadTimeout: 3000
    OkToRetryOnAllOperations: true
    MaxAutoRetries: 1
    MaxAutoRetriesNextServer: 2

💡 说明：开启Sentinel后，Feign调用会被Sentinel监控，可以配置限流、熔断规则。fallback机制仍然有效。

Feign调用流程

flowchart TD A[调用Feign接口方法] --> B[JDK动态代理拦截] B --> C[Contract解析注解] C --> D[RequestInterceptor处理] D --> E[Encoder编码请求] E --> F[Client发送HTTP请求] F --> G[Decoder解码响应] G --> H[返回结果] I[发生异常] --> J{是否有Fallback} J -->|是| K[执行Fallback] J -->|否| L[抛出异常]
  1. 动态代理：Spring启动时为@FeignClient接口创建JDK动态代理
  2. 方法拦截：方法调用时被FeignInvocationHandler拦截
  3. 注解解析：Contract解析@RequestMapping等注解构建请求模板
  4. 拦截器处理：RequestInterceptor统一添加请求头等信息
  5. 编码请求：Encoder将请求对象编码为HTTP请求体
  6. 发送请求：Client（默认HttpURLConnection）发送HTTP请求
  7. 解码响应：Decoder将HTTP响应解码为Java对象
