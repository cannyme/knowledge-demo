Feign声明式HTTP客户端

Feign是Netflix开源的声明式HTTP客户端，通过接口和注解的方式调用远程服务，让HTTP调用像调用本地方法一样简单，实现服务间的优雅通信。

❓

什么是Feign

Feign是声明式REST客户端，通过注解定义接口，自动生成实现类。类似于MyBatis的Mapper接口方式，让调用远程服务像调用本地方法一样简单。

flowchart LR A[服务消费者] -->|Feign调用| B[Feign Client] B -->|HTTP请求| C[服务提供者] C -->|返回数据| B B -->|返回对象| A

Feign vs RestTemplate

对比项RestTemplateFeign
代码风格命令式编程声明式编程
代码量较多（手动拼接URL）简洁（接口+注解）
可读性一般高（像调用本地方法）
维护性URL分散，难维护接口集中，易维护
负载均衡需手动集成Ribbon自动集成
服务降级需手动实现支持Fallback

基本使用

1. 启用Feign

java
@EnableFeignClients
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

2. 定义Feign客户端

java
@FeignClient(
    name = "user-service",           // 服务名
    url = "http://localhost:8081",   // 指定URL（可选）
    fallback = UserServiceFallback.class,  // 降级处理
    configuration = FeignConfig.class      // 自定义配置
)
public interface UserFeignClient {

    @GetMapping("/user/{id}")
    User getUser(@PathVariable("id") Long id);

    @PostMapping("/user")
    User createUser(@RequestBody User user);

    @GetMapping("/users")
    List<User> listUsers(@RequestParam("name") String name);

    @RequestMapping(method = RequestMethod.GET, value = "/search")
    User search(@RequestHeader("X-Token") String token,
                @RequestParam("keyword") String keyword);
}

3. 使用Feign客户端

java
@Service
@RequiredArgsConstructor
public class OrderService {

    private final UserFeignClient userFeignClient;

    public Order createOrder(Long userId) {
        // 像调用本地方法一样调用远程服务
        User user = userFeignClient.getUser(userId);

        Order order = new Order();
        order.setUserId(userId);
        order.setUserName(user.getName());
        return order;
    }
}

服务降级

Fallback实现

java
@Component
public class UserServiceFallback implements UserFeignClient {

    @Override
    public User getUser(Long id) {
        // 返回默认值
        User user = new User();
        user.setId(id);
        user.setName("默认用户");
        return user;
    }

    @Override
    public User createUser(User user) {
        throw new RuntimeException("用户服务不可用");
    }

    @Override
    public List<User> listUsers(String name) {
        return Collections.emptyList();
    }
}

FallbackFactory（可获取异常信息）

java
@Component
public class UserServiceFallbackFactory implements FallbackFactory<UserFeignClient> {

    @Override
    public UserFeignClient create(Throwable cause) {
        log.error("用户服务调用失败", cause);
        return new UserFeignClient() {
            @Override
            public User getUser(Long id) {
                return User.builder()
                    .id(id)
                    .name("服务降级用户")
                    .build();
            }
            // ... 其他方法
        };
    }
}

// 使用FallbackFactory
@FeignClient(name = "user-service", fallbackFactory = UserServiceFallbackFactory.class)

自定义Feign配置

自定义配置类

java
@Configuration
public class FeignConfig {

    // 日志级别
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;
    }

    // 超时配置
    @Bean
    public Request.Options options() {
        return new Request.Options(5000, TimeUnit.MILLISECONDS,
                                   10000, TimeUnit.MILLISECONDS, true);
    }

    // 重试策略
    @Bean
    public Retryer feignRetryer() {
        // 初始间隔100ms，最大间隔1s，最多重试3次
        return new Retryer.Default(100, TimeUnit.SECONDS.toMillis(1), 3);
    }

    // 请求拦截器
    @Bean
    public RequestInterceptor requestInterceptor() {
        return template -> {
            // 添加全局请求头
            template.header("X-Source", "feign");
            template.header("X-Request-Time",
                String.valueOf(System.currentTimeMillis()));
        };
    }
}

请求拦截器

java
@Component
public class AuthFeignInterceptor implements RequestInterceptor {

    @Override
    public void apply(RequestTemplate template) {
        // 从上下文获取Token
        String token = SecurityContextHolder.getContext().getToken();

        // 添加请求头
        template.header("Authorization", "Bearer " + token);

        // 添加查询参数
        template.query("source", "feign");
    }
}

Feign调用流程

flowchart TD A[调用Feign接口方法] --> B[JDK动态代理拦截] B --> C[Contract解析注解] C --> D[RequestInterceptor处理] D --> E[Encoder编码请求] E --> F[Client发送HTTP请求] F --> G[Decoder解码响应] G --> H[返回结果] I[发生异常] --> J{是否有Fallback} J -->|是| K[执行Fallback] J -->|否| L[抛出异常]
  1. 动态代理：Spring启动时为@FeignClient接口创建JDK动态代理
  2. 方法拦截：方法调用时被FeignInvocationHandler拦截
  3. 注解解析：Contract解析@RequestMapping等注解构建请求模板
  4. 拦截器处理：RequestInterceptor统一添加请求头等信息
  5. 编码请求：Encoder将请求对象编码为HTTP请求体
  6. 发送请求：Client（默认HttpURLConnection）发送HTTP请求
  7. 解码响应：Decoder将HTTP响应解码为Java对象

Feign 高级特性：拦截器与日志

java
// 1. 请求拦截器：自动传递 Token（透传认证信息）
@Component
public class FeignAuthInterceptor implements RequestInterceptor {
    @Override
    public void apply(RequestTemplate template) {
        // 从 ThreadLocal 获取当前请求的 token 并透传
        String token = TokenHolder.getToken();
        if (token != null) {
            template.header("Authorization", "Bearer " + token);
        }
        // 透传链路追踪 ID
        String traceId = MDC.get("traceId");
        if (traceId != null) {
            template.header("X-Trace-Id", traceId);
        }
    }
}

// 2. 日志级别配置（开发环境用 FULL，生产用 BASIC 或 NONE）
@Configuration
public class FeignConfig {
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.FULL;  // NONE/BASIC/HEADERS/FULL
    }

    // 自定义错误解码器
    @Bean
    public ErrorDecoder errorDecoder() {
        return (methodKey, response) -> {
            if (response.status() == 404) {
                return new ResourceNotFoundException("远程资源不存在");
            }
            if (response.status() == 503) {
                return new ServiceUnavailableException("远程服务不可用");
            }
            return new FeignException.InternalServerError(
                methodKey, response.request(), null, null);
        };
    }
}

// 3. FeignClient 配置（局部配置优先于全局）
@FeignClient(
    name = "order-service",
    fallbackFactory = OrderFallbackFactory.class,
    configuration = OrderFeignConfig.class  // 局部配置
)
public interface OrderFeignClient {
    @GetMapping("/orders/{id}")
    OrderDTO getOrder(@PathVariable Long id);
}

// Fallback Factory（可获取异常原因）
@Component
public class OrderFallbackFactory implements FallbackFactory<OrderFeignClient> {
    @Override
    public OrderFeignClient create(Throwable cause) {
        log.error("OrderFeignClient 调用失败: {}", cause.getMessage());
        return id -> OrderDTO.empty();  // 降级返回空对象
    }
}

面试高频考点

Q1: Feign 的底层原理是什么？
Feign 基于动态代理实现：①启动时扫描 @FeignClient 注解的接口，通过 JDK 动态代理生成代理类；②调用接口方法时，代理类将方法调用转化为 HTTP 请求（根据 @GetMapping 等注解构建 RequestTemplate）；③通过 FeignBlockingLoadBalancerClient 结合注册中心获取服务实例（负载均衡）；④最终由底层 HTTP 客户端（默认 HttpURLConnection，可替换为 OkHttp/Apache HttpClient）发送请求。
Q2: Feign 与 RestTemplate 的区别？
RestTemplate 是命令式风格，需要手动拼接 URL 和处理响应，代码冗余；Feign 是声明式风格，通过接口+注解定义 HTTP 调用，由框架自动实现，代码更简洁、可读性更好、易于测试（可直接 Mock 接口）。Feign 内置了与 Ribbon/LoadBalancer、Hystrix/Resilience4j 的集成，适合微服务场景；RestTemplate 在 Spring 5 后逐渐被 WebClient 替代。
Q3: Feign 超时配置和重试如何设置？
超时配置（feign.client.config.default.connectTimeout/readTimeout）；重试通过实现 Retryer 接口配置（默认 Retryer.NEVER_RETRY，即不重试）。注意：开启重试时接口必须是幂等的（GET、DELETE、幂等 PUT），非幂等接口（POST）不可开启重试，否则会产生重复数据。推荐结合 Resilience4j CircuitBreaker 做熔断，避免无限重试压垮下游。
Q4: Feign 如何传递请求头（透传 Token）？
①实现 RequestInterceptor 接口，在 apply() 方法中向 RequestTemplate 添加 header（推荐，全局生效）；②在接口方法参数中使用 @RequestHeader 注解（仅对特定方法）；③使用 @Headers 注解在接口类或方法上声明固定 header。注意：异步调用（如 @Async）时 ThreadLocal 中的 Token 会丢失，需在传递前获取并显式传参。

实战陷阱

⚠️
陷阱1：Feign 调用超时却无明确报错 Feign 默认连接超时 10s、读超时 60s，超时后抛出 SocketTimeoutException，但若被 fallback 吞掉，调用方看到的是降级结果而非超时错误，难以排查。应在 FallbackFactory 中记录 cause 的完整堆栈，并配置告警。
⚠️
陷阱2：@SpringQueryMap 遗漏导致 GET 请求参数丢失 Feign 接口用 GET 方法传递复杂对象时，若不加 @SpringQueryMap 注解（或 @QueryMap），参数不会被序列化为 Query String，服务端收不到参数。应对 GET 请求的复杂参数对象加 @SpringQueryMap，或改为一个个独立的 @RequestParam。
⚠️
陷阱3：Feign 日志级别设为 FULL 影响性能 Logger.Level.FULL 会打印完整的请求/响应报文（含 headers 和 body），在高并发场景下会产生大量日志 I/O 开销。生产环境应使用 BASIC（仅记录请求方法、URL、状态码、耗时）或 NONE，仅在排查问题时临时调整为 FULL。
