🌱 SPRING · AOP面向切面

Spring AOP

面向切面编程，将横切关注点（日志、事务、权限等）从业务逻辑中分离出来的强大机制

📖

核心概念

AOP（面向切面编程）：将横切关注点（日志、事务、权限等）从业务逻辑中分离出来，实现代码的解耦和复用。

术语说明
切面 (Aspect)横切关注点的模块化（日志、事务等）
连接点 (JoinPoint)程序执行的特定点（方法调用、异常抛出）
切点 (Pointcut)匹配连接点的表达式
通知 (Advice)在切点执行的动作
目标对象 (Target)被通知的对象
代理 (Proxy)AOP创建的对象，包含通知
织入 (Weaving)将切面应用到目标对象的过程
🎯

应用场景

  • 日志记录 - 统一记录方法入参、返回值、执行时间
  • 事务管理 - 声明式事务控制
  • 权限校验 - 方法级别的权限控制
  • 性能监控 - 统计方法执行耗时
  • 异常处理 - 统一异常捕获和处理
  • 缓存 - 方法结果缓存
🔔

五种通知类型

flowchart TD A[目标方法] --> B[前置通知@Before] B --> C[目标方法执行] C --> D[后置通知@After] D --> E[返回通知@AfterReturning] C -.->|异常| F[异常通知@AfterThrowing] F --> G[方法结束]
通知类型注解执行时机
前置通知@Before方法执行前
后置通知@After方法执行后（无论成功失败）
返回通知@AfterReturning方法成功返回后
异常通知@AfterThrowing方法抛出异常后
环绕通知@Around完全控制方法执行
💡
执行顺序（正常情况）： Around(前置) → Before → 目标方法 → AfterReturning → After → Around(后置)
✂️

切点表达式

execution 表达式语法：

java
execution(修饰符? 返回类型 包名.类名.方法名(参数) 异常?)

常用示例：

java
// 匹配所有public方法
execution(public * *(..))

// 匹配UserService的所有方法
execution(* com.example.service.UserService.*(..))

// 匹配service包下所有类的所有方法
execution(* com.example.service.*.*(..))

// 匹配service包及子包下所有类的所有方法
execution(* com.example.service..*.*(..))

// 匹配所有以save开头的方法
execution(* save*(..))

// 匹配只有一个String参数的方法
execution(* *(String))

其他切点指示符：

  • @annotation() - 匹配有特定注解的方法
  • @within() - 匹配有特定注解的类的方法
  • within() - 匹配类型
  • bean() - 匹配Bean名称
💻

完整示例：日志切面

1. 定义日志注解：

java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Log {
    String value() default "";
}

2. 创建日志切面：

java
@Aspect
@Component
public class LogAspect {

    // 定义切点：所有带@Log注解的方法
    @Pointcut("@annotation(com.example.annotation.Log)")
    public void logPointcut() {}

    // 环绕通知（最常用）
    @Around("logPointcut()")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        String methodName = joinPoint.getSignature().getName();

        System.out.println("[开始] " + methodName);

        try {
            Object result = joinPoint.proceed();
            System.out.println("[成功] " + methodName + " 耗时: " +
                (System.currentTimeMillis() - start) + "ms");
            return result;
        } catch (Exception e) {
            System.out.println("[异常] " + methodName + " " + e.getMessage());
            throw e;
        }
    }
}

3. 使用切面：

java
@Service
public class UserService {

    @Log("保存用户")
    public void saveUser(User user) {
        // 业务逻辑
    }
}

JDK动态代理 vs CGLIB

特性JDK动态代理CGLIB
原理基于接口基于继承
限制目标类必须实现接口不能代理final类/方法
性能JDK8后效率接近CGLIB生成子类，略快
Spring默认有接口时使用无接口时使用
💡
配置方式： spring.aop.proxy-target-class=true - 强制使用CGLIB
spring.aop.proxy-target-class=false - 有接口用JDK，无接口用CGLIB
⚠️

常见坑点

⚠️
1. 自调用问题 同一个类中，方法A调用方法B，B的切面不生效。原因：自调用走的是this，不是代理对象。
解决：使用 AopContext.currentProxy() 获取代理对象
⚠️
2. private方法 AOP无法代理private方法（CGLIB限制）
解决：改为protected或public
⚠️
3. final方法/类 CGLIB无法代理final，会静默忽略
解决：去掉final修饰符，或使用接口+JDK代理
💡
4. 循环依赖 + AOP Spring通过三级缓存解决循环依赖问题。早期暴露的Bean可能已经是代理对象，确保AOP能正确应用。

@Around 实战：接口限流切面

使用 AOP + 自定义注解实现接口限流，是生产中的高频场景。

java
// 1. 限流注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RateLimit {
    int qps() default 100;          // 每秒允许请求数
    String key() default "";        // 限流key（支持SpEL）
}

// 2. 限流切面
@Aspect
@Component
public class RateLimitAspect {

    private final Map<String, RateLimiter> limiters = new ConcurrentHashMap<>();

    @Around("@annotation(rateLimit)")
    public Object limit(ProceedingJoinPoint pjp, RateLimit rateLimit) throws Throwable {
        String key = buildKey(pjp, rateLimit.key());
        RateLimiter limiter = limiters.computeIfAbsent(
            key, k -> RateLimiter.create(rateLimit.qps())
        );

        if (!limiter.tryAcquire(50, TimeUnit.MILLISECONDS)) {
            throw new RuntimeException("请求过于频繁，请稍后重试");
        }
        return pjp.proceed();
    }

    private String buildKey(ProceedingJoinPoint pjp, String keyExpr) {
        // 解析SpEL表达式获取动态key
        MethodSignature sig = (MethodSignature) pjp.getSignature();
        return sig.getDeclaringTypeName() + "." + sig.getName() + ":" + keyExpr;
    }
}

// 3. 使用
@RestController
public class OrderController {
    @RateLimit(qps = 50, key = "#userId")
    @GetMapping("/orders")
    public List<Order> getOrders(String userId) {
        return orderService.findByUser(userId);
    }
}

面试高频考点

Q1: JDK动态代理和CGLIB的区别？Spring Boot默认使用哪种？
JDK动态代理基于接口（java.lang.reflect.Proxy），目标类必须实现接口；CGLIB基于继承（生成子类字节码），不要求接口，但不能代理final类/方法。Spring Boot 2.x 默认使用 CGLIB（spring.aop.proxy-target-class=true），可通过配置切换。JDK 8+ 后两者性能差距已很小。
Q2: AOP 通知的执行顺序是什么？多个切面如何控制顺序？
单切面正常：Around前置 → Before → 目标方法 → AfterReturning → After → Around后置；异常时：Around前置 → Before → 目标方法(异常) → AfterThrowing → After → Around异常传播。多切面通过 @Order 注解控制（值越小优先级越高），形成切面链（责任链模式）。
Q3: AOP 自调用为什么会失效？如何解决？
AOP 通过代理实现，外部调用走代理对象；同类内部 this.method() 调用绕过代理直接调用目标对象，切面不生效。解决方案：①注入自身Bean（@Autowired private MyService self）②AopContext.currentProxy()获取代理③拆分为两个类④使用 AspectJ 编译时织入（无代理限制）。
Q4: @Transactional 是如何通过 AOP 实现的？
@Transactional 底层是 TransactionInterceptor（实现了 MethodInterceptor），作为 Around 通知工作：方法前开启事务，正常返回则提交，抛出 RuntimeException 则回滚。因此 @Transactional 也受 AOP 自调用失效的影响，同类中调用标注事务的方法，事务不生效。
⚡

实战陷阱

⚠️
陷阱1：@Transactional 标注在 private 方法上无效 CGLIB 代理无法重写 private 方法，事务切面不会拦截。Spring 5.x 会在启动时打印警告，但不报错。务必将需要事务的方法改为 public。
⚠️
陷阱2：ProceedingJoinPoint.proceed() 必须调用 @Around 通知中如果忘记调用 pjp.proceed()，目标方法不会执行。返回 null 时调用方还会得到 null 返回值，极难排查。务必确保调用 proceed() 并返回其结果。
⚠️
陷阱3：切点表达式匹配范围过宽导致性能问题 execution(* *.*(..)) 会匹配所有方法（包括 toString、hashCode 等），产生大量无谓的代理调用。应收窄匹配范围，使用 @annotation 精确匹配，或限定包路径 execution(* com.example.service.*.*(..))。
