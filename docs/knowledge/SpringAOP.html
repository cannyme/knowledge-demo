🌱 SPRING

Spring AOP

面向切面编程，将横切关注点（日志、事务、权限等）从业务逻辑中分离出来的强大机制

📖

核心概念

AOP（面向切面编程）：将横切关注点（日志、事务、权限等）从业务逻辑中分离出来，实现代码的解耦和复用。

术语说明
切面 (Aspect)横切关注点的模块化（日志、事务等）
连接点 (JoinPoint)程序执行的特定点（方法调用、异常抛出）
切点 (Pointcut)匹配连接点的表达式
通知 (Advice)在切点执行的动作
目标对象 (Target)被通知的对象
代理 (Proxy)AOP创建的对象，包含通知
织入 (Weaving)将切面应用到目标对象的过程
🎯

应用场景

  • 日志记录 - 统一记录方法入参、返回值、执行时间
  • 事务管理 - 声明式事务控制
  • 权限校验 - 方法级别的权限控制
  • 性能监控 - 统计方法执行耗时
  • 异常处理 - 统一异常捕获和处理
  • 缓存 - 方法结果缓存
🔔

五种通知类型

flowchart TD A[目标方法] --> B[前置通知@Before] B --> C[目标方法执行] C --> D[后置通知@After] D --> E[返回通知@AfterReturning] C -.->|异常| F[异常通知@AfterThrowing] F --> G[方法结束]
通知类型注解执行时机
前置通知@Before方法执行前
后置通知@After方法执行后（无论成功失败）
返回通知@AfterReturning方法成功返回后
异常通知@AfterThrowing方法抛出异常后
环绕通知@Around完全控制方法执行
执行顺序（正常情况）：
Around(前置) → Before → 目标方法 → AfterReturning → After → Around(后置)
✂️

切点表达式

execution 表达式语法：

java
execution(修饰符? 返回类型 包名.类名.方法名(参数) 异常?)

常用示例：

java
// 匹配所有public方法
execution(public * *(..))

// 匹配UserService的所有方法
execution(* com.example.service.UserService.*(..))

// 匹配service包下所有类的所有方法
execution(* com.example.service.*.*(..))

// 匹配service包及子包下所有类的所有方法
execution(* com.example.service..*.*(..))

// 匹配所有以save开头的方法
execution(* save*(..))

// 匹配只有一个String参数的方法
execution(* *(String))

其他切点指示符：

  • @annotation() - 匹配有特定注解的方法
  • @within() - 匹配有特定注解的类的方法
  • within() - 匹配类型
  • bean() - 匹配Bean名称
💻

完整示例：日志切面

1. 定义日志注解：

java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Log {
    String value() default "";
}

2. 创建日志切面：

java
@Aspect
@Component
public class LogAspect {

    // 定义切点：所有带@Log注解的方法
    @Pointcut("@annotation(com.example.annotation.Log)")
    public void logPointcut() {}

    // 环绕通知（最常用）
    @Around("logPointcut()")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        String methodName = joinPoint.getSignature().getName();

        System.out.println("[开始] " + methodName);

        try {
            Object result = joinPoint.proceed();
            System.out.println("[成功] " + methodName + " 耗时: " +
                (System.currentTimeMillis() - start) + "ms");
            return result;
        } catch (Exception e) {
            System.out.println("[异常] " + methodName + " " + e.getMessage());
            throw e;
        }
    }
}

3. 使用切面：

java
@Service
public class UserService {

    @Log("保存用户")
    public void saveUser(User user) {
        // 业务逻辑
    }
}

JDK动态代理 vs CGLIB

特性JDK动态代理CGLIB
原理基于接口基于继承
限制目标类必须实现接口不能代理final类/方法
性能JDK8后效率接近CGLIB生成子类，略快
Spring默认有接口时使用无接口时使用
配置方式：
spring.aop.proxy-target-class=true - 强制使用CGLIB
spring.aop.proxy-target-class=false - 有接口用JDK，无接口用CGLIB
⚠️

常见坑点

1. 自调用问题
同一个类中，方法A调用方法B，B的切面不生效。原因：自调用走的是this，不是代理对象。
解决：使用 AopContext.currentProxy() 获取代理对象
2. private方法
AOP无法代理private方法（CGLIB限制）
解决：改为protected或public
3. final方法/类
CGLIB无法代理final，会静默忽略
解决：去掉final修饰符，或使用接口+JDK代理
4. 循环依赖 + AOP
Spring通过三级缓存解决循环依赖问题。早期暴露的Bean可能已经是代理对象，确保AOP能正确应用。
