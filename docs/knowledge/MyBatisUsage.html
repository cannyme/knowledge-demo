MyBatis核心知识点

MyBatis是一款优秀的半自动ORM框架，通过XML或注解方式将Java对象与SQL语句映射，提供灵活的SQL控制和强大的动态SQL支持。

⚖️

MyBatis vs Hibernate/JPA

特性MyBatisHibernate/JPA
ORM类型半自动ORM全自动ORM
SQL控制完全控制，手写SQL自动生成，HQL/JPQL
学习曲线较低，SQL开发者友好较高，需要理解ORM概念
复杂查询优秀，原生SQL支持一般，需要多表关联配置
缓存机制一级、二级缓存一级、二级、查询缓存
适用场景复杂查询、遗留系统快速开发、简单CRUD
选型建议：
  • 需要优化SQL、复杂查询、遗留数据库 → 选择MyBatis
  • 快速开发、对象驱动、简单CRUD → 选择JPA/Hibernate
  • 混合使用：MyBatis处理复杂查询，JPA处理简单CRUD

Mapper XML配置

resultMap用于自定义结果集映射，解决列名与属性名不一致问题

XML
<resultMap id="UserResultMap" type="com.example.entity.User">
    <id property="id" column="user_id"/>
    <result property="name" column="user_name"/>
    <result property="email" column="user_email"/>
    <result property="createTime" column="create_time"/>
</resultMap>

<select id="selectById" resultMap="UserResultMap">
    SELECT user_id, user_name, user_email, create_time
    FROM user WHERE user_id = #{id}
</select>
💡 开启驼峰映射

在mybatis-config.xml中配置：<setting name="mapUnderscoreToCamelCase" value="true"/>

可自动将create_time映射为createTime，简化resultMap配置

动态SQL标签：if、where、foreach、choose、set

XML
<!-- where标签自动处理AND/OR -->
<select id="selectByCondition" resultType="User">
    SELECT * FROM user
    <where>
        <if test="name != null and name != ''">
            AND name LIKE CONCAT('%', #{name}, '%')
        </if>
        <if test="email != null">
            AND email = #{email}
        </if>
        <if test="status != null">
            AND status = #{status}
        </if>
    </where>
</select>

<!-- foreach批量操作 -->
<select id="selectByIds" resultType="User">
    SELECT * FROM user WHERE id IN
    <foreach collection="ids" item="id" open="(" separator="," close=")">
        #{id}
    </foreach>
</select>

<!-- choose多条件选择 -->
<select id="selectByPriority" resultType="User">
    SELECT * FROM user WHERE 1=1
    <choose>
        <when test="id != null">AND id = #{id}</when>
        <when test="name != null">AND name = #{name}</when>
        <otherwise>AND status = 1</otherwise>
    </choose>
</select>

<!-- set标签自动处理逗号 -->
<update id="updateSelective">
    UPDATE user
    <set>
        <if test="name != null">name = #{name},</if>
        <if test="email != null">email = #{email},</if>
        <if test="phone != null">phone = #{phone},</if>
    </set>
    WHERE id = #{id}
</update>

association一对一，collection一对多

XML
<resultMap id="UserWithDeptMap" type="User">
    <id property="id" column="user_id"/>
    <result property="name" column="user_name"/>
    <!-- 一对一关联 -->
    <association property="department" javaType="Department">
        <id property="deptId" column="dept_id"/>
        <result property="deptName" column="dept_name"/>
    </association>
</resultMap>

<resultMap id="UserWithOrdersMap" type="User">
    <id property="id" column="user_id"/>
    <result property="name" column="user_name"/>
    <!-- 一对多关联 -->
    <collection property="orders" ofType="Order">
        <id property="orderId" column="order_id"/>
        <result property="orderNo" column="order_no"/>
        <result property="amount" column="amount"/>
    </collection>
</resultMap>

注解方式

Java
@Mapper
public interface UserMapper {

    @Select("SELECT * FROM user WHERE id = #{id}")
    User selectById(Long id);

    @Select("SELECT * FROM user WHERE name = #{name}")
    @Results({
        @Result(property = "id", column = "id"),
        @Result(property = "name", column = "name"),
        @Result(property = "orders", column = "id",
                many = @Many(select = "selectOrdersByUserId"))
    })
    User selectWithOrders(String name);

    @Insert("INSERT INTO user(name, email) VALUES(#{name}, #{email})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(User user);

    @Update("UPDATE user SET name = #{name} WHERE id = #{id}")
    int update(User user);

    @Delete("DELETE FROM user WHERE id = #{id}")
    int delete(Long id);

    @SelectProvider(type = UserSqlProvider.class, method = "selectByCondition")
    List<User> selectByCondition(UserCondition condition);
}
XML vs 注解选择：
  • 简单SQL、快速开发 → 注解方式
  • 复杂SQL、动态SQL → XML方式（推荐）
  • 混合使用：简单用注解，复杂用XML

#{} vs ${} 区别

特性#{}${}
处理方式预编译，使用占位符?字符串直接替换
SQL注入风险安全，防止注入不安全，有注入风险
适用场景参数值（推荐）表名、列名、动态排序
示例WHERE id = #{id}ORDER BY ${column}
⚠️ SQL注入风险

错误示例：WHERE name = '${name}'

如果name传入：' OR '1'='1，将导致全表数据泄露

正确使用：WHERE name = #{name} → 预编译为 WHERE name = ?

Java
// 安全的动态表名查询
@Select("SELECT * FROM ${tableName} WHERE id = #{id}")
User selectByTable(@Param("tableName") String tableName, @Param("id") Long id);

// 使用前必须校验表名，防止注入
public User safeSelect(String tableName, Long id) {
    // 白名单校验
    if (!Arrays.asList("user", "order", "product").contains(tableName)) {
        throw new IllegalArgumentException("非法表名");
    }
    return mapper.selectByTable(tableName, id);
}

插件机制 Interceptor

MyBatis插件可以拦截四大对象：Executor、StatementHandler、ParameterHandler、ResultSetHandler

Java
@Intercepts({
    @Signature(type = Executor.class, method = "query",
        args = {MappedStatement.class, Object.class, RowBounds.class, ResultHandler.class})
})
public class SqlCostInterceptor implements Interceptor {
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        long startTime = System.currentTimeMillis();
        try {
            return invocation.proceed();
        } finally {
            long cost = System.currentTimeMillis() - startTime;
            MappedStatement ms = (MappedStatement) invocation.getArgs()[0];
            System.out.println("SQL [" + ms.getId() + "] 执行耗时: " + cost + "ms");
        }
    }
}

// 配置插件
@Bean
public SqlCostInterceptor sqlCostInterceptor() {
    return new SqlCostInterceptor();
}
常见插件功能
PageHelper分页插件，自动处理分页逻辑
SQL日志打印执行的SQL和参数
数据权限自动添加数据权限过滤条件
性能监控SQL执行时间统计

XML
<!-- 开启二级缓存 -->
<settings>
    <setting name="cacheEnabled" value="true"/>
</settings>

<!-- Mapper中启用缓存 -->
<cache eviction="LRU" flushInterval="60000" size="512" readOnly="true"/>

<!-- 禁用缓存 -->
<select id="selectLatest" useCache="false">

⚠️ 分布式缓存注意事项

二级缓存默认是本地缓存，分布式环境下会导致数据不一致。建议使用Redis作为二级缓存实现。

批量操作优化

XML
<insert id="batchInsert" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO user (name, email) VALUES
    <foreach collection="list" item="user" separator=",">
        (#{user.name}, #{user.email})
    </foreach>
</insert>

// 生成SQL：INSERT INTO user (name, email) VALUES ('a','a@mail'),('b','b@mail')
💡 注意

单次插入数据量过大可能导致SQL过长，建议分批处理，每批1000条左右

Java
// 使用Batch Executor
SqlSession session = sqlSessionFactory.openSession(ExecutorType.BATCH);
try {
    UserMapper mapper = session.getMapper(UserMapper.class);
    for (User user : userList) {
        mapper.insert(user);
    }
    session.commit();
} finally {
    session.close();
}

// Spring Boot配置
@Bean
public SqlSessionTemplate sqlSessionTemplate(SqlSessionFactory factory) {
    return new SqlSessionTemplate(factory, ExecutorType.BATCH);
}
💡 Batch Executor优势
  • 重用预编译语句，减少SQL解析开销
  • 批量提交，减少网络往返
  • 性能比foreach更好（大数据量）
方式适用场景性能复杂度
foreach小到中等数据量（<1000条）良好简单
Batch Executor大数据量（>1000条）优秀中等
单条循环不推荐差简单
