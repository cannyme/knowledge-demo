模板方法模式

定义一个操作中的算法骨架，而将一些步骤延迟到子类中。模板方法使得子类可以不改变一个算法的结构即可重定义该算法的某些特定步骤。

🎯

模式定义与意图

模板方法模式（Template Method Pattern）是一种行为型设计模式，它在父类中定义一个算法的骨架，将某些步骤延迟到子类实现。模板方法让子类在不改变算法结构的情况下，重新定义算法的某些步骤。

核心思想：定义算法执行的固定步骤（骨架），其中部分步骤由抽象方法表示，子类通过实现这些抽象方法来定制算法的具体行为。

生活中的类比：

  • 银行开户流程：填写资料 -> 身份验证 -> 开设账户 -> 发放卡片（各银行验证方式不同）
  • 泡茶/咖啡：烧水 -> 冲泡 -> 倒入杯中 -> 添加调料（茶和咖啡冲泡方式不同）
  • 简历模板：个人信息 -> 教育背景 -> 工作经历 -> 技能（各部分内容不同）
🏗️

架构原理

classDiagram class AbstractClass { +templateMethod() +primitiveOperation1()* +primitiveOperation2()* #hook() +concreteOperation() } class ConcreteClass1 { +primitiveOperation1() +primitiveOperation2() } class ConcreteClass2 { +primitiveOperation1() +primitiveOperation2() } class Client AbstractClass <|-- ConcreteClass1 AbstractClass <|-- ConcreteClass2 Client --> AbstractClass

角色说明：

角色说明
AbstractClass抽象类，定义模板方法和基本方法
ConcreteClass具体子类，实现抽象方法，完成特定步骤
templateMethod模板方法，定义算法骨架，final修饰防止重写
primitiveOperation抽象操作，子类必须实现
hook钩子方法，子类可选择性覆盖
concreteOperation具体方法，子类直接使用或覆盖

代码实现

基础模板方法实现

Java
// 抽象模板类
abstract class DataProcessor {
    /**
     * 模板方法：定义算法骨架（final防止子类覆盖）
     */
    public final void process(String data) {
        // 1. 验证数据
        if (!validate(data)) {
            System.out.println("数据验证失败，终止处理");
            return;
        }

        // 2. 解析数据
        Object parsedData = parse(data);

        // 3. 处理数据（具体实现由子类决定）
        Object result = doProcess(parsedData);

        // 4. 保存结果
        save(result);

        // 5. 记录日志（钩子方法，子类可选实现）
        log();
    }

    // 具体方法：通用实现
    protected boolean validate(String data) {
        if (data == null || data.isEmpty()) {
            System.out.println("数据为空");
            return false;
        }
        System.out.println("数据验证通过");
        return true;
    }

    protected Object parse(String data) {
        System.out.println("解析数据：" + data);
        return data;
    }

    // 抽象方法：子类必须实现
    protected abstract Object doProcess(Object data);

    // 具体方法：通用实现
    protected void save(Object result) {
        System.out.println("保存结果：" + result);
    }

    // 钩子方法：子类可选实现
    protected void log() {
        // 默认空实现
    }
}

// JSON处理器
class JsonDataProcessor extends DataProcessor {
    @Override
    protected Object doProcess(Object data) {
        System.out.println("处理JSON数据：" + data);
        return "{\"processed\": \"" + data + "\"}";
    }

    @Override
    protected void log() {
        System.out.println("[JSON处理器] 记录处理日志");
    }
}

// XML处理器
class XmlDataProcessor extends DataProcessor {
    @Override
    protected Object doProcess(Object data) {
        System.out.println("处理XML数据：" + data);
        return "<processed>" + data + "</processed>";
    }

    @Override
    protected Object parse(String data) {
        System.out.println("特殊XML解析：" + data);
        return data.toUpperCase();
    }
}

订单处理模板

Java
// 订单处理模板
abstract class OrderProcessor {
    // 模板方法
    public final void processOrder(Order order) {
        // 1. 验证订单
        validateOrder(order);

        // 2. 计算价格
        double totalPrice = calculatePrice(order);
        order.setTotalPrice(totalPrice);

        // 3. 扣减库存
        deductStock(order);

        // 4. 支付（抽象步骤）
        boolean paid = doPayment(order);
        if (!paid) {
            rollback(order);
            return;
        }

        // 5. 创建物流
        createShipping(order);

        // 6. 发送通知（钩子）
        sendNotification(order);
    }

    protected void validateOrder(Order order) {
        if (order.getItems() == null || order.getItems().isEmpty()) {
            throw new RuntimeException("订单商品为空");
        }
        System.out.println("订单验证通过");
    }

    protected double calculatePrice(Order order) {
        return order.getItems().stream()
            .mapToDouble(Item::getPrice)
            .sum();
    }

    protected void deductStock(Order order) {
        System.out.println("扣减库存");
    }

    // 抽象方法：不同订单类型不同支付方式
    protected abstract boolean doPayment(Order order);

    protected void rollback(Order order) {
        System.out.println("订单回滚");
    }

    protected void createShipping(Order order) {
        System.out.println("创建物流");
    }

    // 钩子方法
    protected void sendNotification(Order order) {
        // 默认不发送
    }
}

// 普通订单
class NormalOrderProcessor extends OrderProcessor {
    @Override
    protected boolean doPayment(Order order) {
        System.out.println("普通订单支付：" + order.getTotalPrice() + "元");
        return true;
    }
}

// VIP订单
class VipOrderProcessor extends OrderProcessor {
    @Override
    protected double calculatePrice(Order order) {
        double basePrice = super.calculatePrice(order);
        return basePrice * 0.9;  // 9折优惠
    }

    @Override
    protected boolean doPayment(Order order) {
        System.out.println("VIP订单支付：" + order.getTotalPrice() + "元（已优惠）");
        return true;
    }

    @Override
    protected void sendNotification(Order order) {
        System.out.println("发送VIP专属通知");
    }
}

回调方式（更灵活）

模板方法使用继承，回调使用组合。Java 8+使用Lambda更简洁。

Java
import java.util.function.*;

class DataProcessorWithCallback {
    public void process(String data,
                       Function<String, Object> parser,
                       Function<Object, Object> processor) {
        // 1. 验证
        if (data == null || data.isEmpty()) {
            return;
        }

        // 2. 解析（回调）
        Object parsed = parser.apply(data);

        // 3. 处理（回调）
        Object result = processor.apply(parsed);

        // 4. 保存
        System.out.println("保存结果：" + result);
    }
}

// 使用Lambda
DataProcessorWithCallback callbackProcessor = new DataProcessorWithCallback();
callbackProcessor.process(
    "test data",
    s -> "解析后：" + s,
    o -> "处理后：" + o
);

// 对比：JDBCTemplate使用回调
callbackProcessor.process(
    "SELECT * FROM user WHERE id = ?",
    sql -> createPreparedStatement(sql),  // 解析
    stmt -> executeQuery(stmt)            // 处理
);

框架中的应用

1. HttpServlet

Servlet的service()方法是经典的模板方法：

Java
// HttpServlet中的模板方法
public abstract class HttpServlet extends GenericServlet {
    // 模板方法
    protected void service(HttpServletRequest req, HttpServletResponse resp) {
        String method = req.getMethod();
        switch (method) {
            case "GET":
                doGet(req, resp);   // 钩子方法
                break;
            case "POST":
                doPost(req, resp);  // 钩子方法
                break;
            case "PUT":
                doPut(req, resp);
                break;
            case "DELETE":
                doDelete(req, resp);
                break;
            // ...
        }
    }

    // 钩子方法：默认空实现
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) {
        // 默认返回405错误
    }
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) {
        // 默认返回405错误
    }
}

// 开发者只需覆盖需要的方法
public class MyServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) {
        // 处理GET请求
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) {
        // 处理POST请求
    }
}

2. Spring JdbcTemplate

Java
// JdbcTemplate使用模板方法模式
public class JdbcTemplate {
    public <T> T query(String sql, RowMapper<T> rowMapper, Object... params) {
        // 1. 获取连接（固定）
        Connection conn = getConnection();
        try {
            // 2. 创建语句（固定）
            PreparedStatement stmt = createStatement(conn, sql, params);

            // 3. 执行查询（固定）
            ResultSet rs = stmt.executeQuery();

            // 4. 映射结果（变化部分，由调用者实现）
            T result = rowMapper.mapRow(rs);

            // 5. 返回结果（固定）
            return result;
        } finally {
            // 6. 关闭连接（固定）
            closeConnection(conn);
        }
    }
}

// 使用
User user = jdbcTemplate.query(
    "SELECT * FROM user WHERE id = ?",
    (rs, rowNum) -> {  // RowMapper回调
        User u = new User();
        u.setId(rs.getLong("id"));
        u.setName(rs.getString("name"));
        return u;
    },
    userId
);

适用场景

  • 流程固定：算法骨架固定，但某些步骤的具体实现不同
  • 代码复用：多个类有共同的行为，只在特定步骤上有差异
  • 控制子类扩展：只允许在特定点扩展行为（钩子方法）
  • 框架设计：为框架提供扩展点，子类实现具体逻辑
💡 常见应用场景
  • 数据处理流程（解析 -> 转换 -> 存储）
  • 业务流程（审批、支付、订单处理）
  • 测试框架（JUnit的setUp/test/tearDown）
  • IO流（InputStream的read模板方法）
🔄

与策略模式对比

维度模板方法模式策略模式
复用级别代码级别（继承）算法级别（组合）
控制方式父类控制流程，子类实现细节客户端选择算法
扩展方式继承抽象类实现策略接口
依赖关系子类依赖父类上下文依赖策略
适用场景算法骨架固定，部分步骤变化多个可互换的算法
灵活性较低（继承是强耦合）较高（运行时切换策略）
选择建议：如果算法骨架固定，使用模板方法；如果需要完全替换算法或运行时切换，使用策略模式。
⚖️

优缺点分析

✅ 优点
  • 代码复用：将公共代码放在父类，减少重复
  • 反向控制：好莱坞原则"Don't call us, we'll call you"
  • 符合开闭原则：通过扩展子类而非修改父类来增加功能
  • 统一流程：确保算法骨架的一致性
❌ 缺点
  • 类数量增加：每个变体都需要一个子类
  • 继承限制：Java单继承限制了扩展性
  • 复杂性：模板方法和钩子方法设计需要仔细考虑
  • 违反里氏替换：父类修改可能影响所有子类
📋

最佳实践

设计要点：
  • 模板方法使用final修饰，防止子类重写算法骨架
  • 抽象方法必须是protected，不需要对外暴露
  • 钩子方法可以提供默认空实现，让子类选择是否覆盖
  • 尽量减少具体方法的数量，保持模板类的简洁
💡 好莱坞原则

"Don't call us, we'll call you" - 父类决定何时调用子类的方法，而不是子类主动调用父类。这是一种反向控制，让高层组件决定何时、如何调用低层组件。
