Docker与Kubernetes基础

Docker是容器化技术的代表，Kubernetes是容器编排的行业标准。掌握这两个技术对于现代云原生应用开发和部署至关重要。

🐳

Docker核心概念

概念说明类比
Image（镜像）只读模板，包含运行应用所需的一切类（Class）
Container（容器）镜像的运行实例对象（Object）
Dockerfile构建镜像的脚本构建脚本
Registry（仓库）存储和分发镜像（Docker Hub）Maven仓库
Volume（卷）持久化存储，独立于容器生命周期外部硬盘
Network（网络）容器间通信机制虚拟网络

Dockerfile指令详解

指令说明示例
FROM基础镜像FROM openjdk:17-jdk-slim
WORKDIR工作目录WORKDIR /app
COPY复制文件COPY target/*.jar app.jar
ADD复制（支持URL、自动解压）ADD app.tar.gz /app
RUN执行命令RUN apt-get update && apt-get install -y curl
ENV环境变量ENV JAVA_OPTS="-Xmx512m"
EXPOSE暴露端口EXPOSE 8080
ENTRYPOINT容器启动命令（不可覆盖）ENTRYPOINT ["java","-jar","app.jar"]
CMD容器启动命令（可被覆盖）CMD ["--server.port=8080"]
VOLUME数据卷VOLUME /app/logs

Spring Boot应用Dockerfile示例

Dockerfile
# 多阶段构建：构建阶段
FROM maven:3.8-openjdk-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# 运行阶段
FROM openjdk:17-jdk-slim
WORKDIR /app

# 从构建阶段复制jar
COPY --from=build /app/target/*.jar app.jar

# 创建非root用户
RUN useradd -m -u 1000 appuser
USER appuser

# 暴露端口
EXPOSE 8080

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost:8080/actuator/health || exit 1

# 启动命令
ENTRYPOINT ["java", "-jar", "app.jar"]
💡 多阶段构建优势
  • 减小镜像体积（不包含Maven等构建工具）
  • 提高安全性（不暴露源码和构建工具）
  • 构建和运行分离，职责清晰

Docker常用命令

Shell
# 构建镜像
docker build -t myapp:1.0 .

# 查看镜像
docker images
docker rmi myapp:1.0

# 推送/拉取镜像
docker push registry/myapp:1.0
docker pull nginx:latest

# 导出/导入镜像
docker save -o myapp.tar myapp:1.0
docker load -i myapp.tar
Shell
# 运行容器
docker run -d -p 8080:8080 --name myapp myapp:1.0

# 常用选项
# -d: 后台运行
# -p: 端口映射
# -v: 挂载卷
# -e: 环境变量
# --network: 指定网络
# --restart: 重启策略

# 容器管理
docker ps              # 查看运行中容器
docker ps -a           # 查看所有容器
docker stop myapp      # 停止容器
docker start myapp     # 启动容器
docker rm myapp        # 删除容器
docker logs -f myapp   # 查看日志
docker exec -it myapp /bin/bash  # 进入容器
YAML
version: '3.8'
services:

    ports:

      - SPRING_PROFILES_ACTIVE=prod

    depends_on:
      - db
      - redis

  redis:
    image: redis:7-alpine
    volumes:
      - redis-data:/data
    networks:
      - app-network

volumes:
  db-data:
  redis-data:

networks:
  app-network:
    driver: bridge
Shell

# 查看服务状态

K8s部署YAML示例

YAML
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
  namespace: default
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myapp:1.0
        ports:
        - containerPort: 8080
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        env:
        - name: SPRING_PROFILES_ACTIVE
          valueFrom:
            configMapKeyRef:
              name: myapp-config
              key: profile
        livenessProbe:
          httpGet:
            path: /actuator/health
            port: 8080
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /actuator/health
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 5

kind: Service

  - host: myapp.example.com

kubectl常用命令

Shell
# 查看资源
kubectl get pods                    # 查看Pod
kubectl get pods -o wide            # 详细信息（节点IP）
kubectl get deployments             # 查看Deployment

# 查看详情
kubectl describe pod myapp-xxx      # Pod详情

kubectl exec -it myapp-xxx -- /bin/bash  # 进入容器

kubectl apply -f deployment.yaml    # 应用配置
kubectl delete -f deployment.yaml   # 删除资源

kubectl scale deployment myapp --replicas=5

kubectl set image deployment/myapp myapp=myapp:2.0
kubectl rollout status deployment/myapp
kubectl rollout undo deployment/myapp  # 回滚

# 端口转发
kubectl port-forward pod/myapp-xxx 8080:8080

💡 负载均衡算法

Kube-proxy默认使用iptables模式，支持随机或轮询负载均衡。新版本支持IPVS模式，提供更多负载均衡算法。

kubectl rollout resume deployment/myapp

kubectl rollout history deployment/myapp

kubectl rollout undo deployment/myapp

# 回滚到指定版本
kubectl rollout undo deployment/myapp --to-revision=2

Docker vs Kubernetes关系

维度DockerKubernetes
定位容器运行时容器编排平台
管理范围单机容器集群容器
功能构建、运行、分发调度、扩缩、自愈、服务发现
关系K8s使用Docker作为容器运行时（CRI）
使用场景：
  • 开发环境：Docker + Docker Compose 足够
  • 测试环境：可以使用Minikube或单节点K8s
  • 生产环境：Kubernetes是必须的，处理多节点、高可用、自动扩缩
✅

最佳实践

💡 Docker最佳实践
  • 使用多阶段构建减小镜像体积
  • 使用.dockerignore排除不必要文件
  • 不要以root用户运行容器
  • 使用特定版本标签，不使用latest
  • 利用缓存优化构建顺序（不常变动的放前面）
  • 添加健康检查HEALTHCHECK
💡 K8s最佳实践
  • 配置资源限制（requests/limits）
  • 配置健康检查（liveness/readiness）
  • 使用ConfigMap/Secret管理配置
  • 使用Helm管理复杂应用
  • 配置HPA自动扩缩容
  • 使用Namespace隔离环境
  • 设置Pod安全策略
