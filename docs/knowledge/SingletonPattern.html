单例模式

确保一个类只有一个实例，并提供一个全局访问点。单例模式是创建型设计模式中最常用的一种，广泛应用于配置管理、数据库连接池、日志记录等场景。

📖

模式定义与意图

定义：单例模式（Singleton Pattern）是一种创建型设计模式，它确保一个类只有一个实例存在，并提供一个全局访问点来获取该实例。

核心思想：通过私有化构造方法，控制实例的创建过程，保证整个应用程序中只存在一个实例对象。

classDiagram class Singleton { -static Singleton instance -Singleton() +static Singleton getInstance() }

关键要素：

  • 私有化构造器：防止外部通过new关键字创建实例
  • 静态实例引用：持有唯一的实例引用
  • 全局访问点：提供公共静态方法获取实例
🎯

适用场景

以下场景适合使用单例模式：

  • 配置管理器：应用配置信息只需加载一次，全局共享
  • 数据库连接池：管理有限的数据库连接资源
  • 日志记录器：统一处理日志输出，避免重复创建
  • 缓存管理器：全局缓存数据的统一管理
  • 线程池：管理系统线程资源的池化对象
💻

代码实现

饿汉式（Eager Initialization） - 类加载时就创建实例，线程安全但不支持懒加载

Java
public class EagerSingleton {
    // 类加载时就创建实例
    private static final EagerSingleton INSTANCE = new EagerSingleton();

    // 私有构造方法，防止外部实例化
    private EagerSingleton() {}

    public static EagerSingleton getInstance() {
        return INSTANCE;
    }
}

懒汉式（Lazy Initialization） - 支持懒加载但线程不安全（不推荐单独使用）

Java
public class LazySingleton {
    private static LazySingleton instance;

    private LazySingleton() {}

    // 线程不安全：多线程可能同时进入if判断
    public static LazySingleton getInstance() {
        if (instance == null) {
            instance = new LazySingleton();
        }
        return instance;
    }

    // 线程安全版本（同步方法）- 性能较差
    public static synchronized LazySingleton getInstanceSafe() {
        if (instance == null) {
            instance = new LazySingleton();
        }
        return instance;
    }
}

双重检查锁（Double Checked Locking） - 线程安全 + 懒加载 + 高性能

Java
public class DCLSingleton {
    // volatile 禁止指令重排序
    private static volatile DCLSingleton instance;

    private DCLSingleton() {}

    public static DCLSingleton getInstance() {
        // 第一次检查：避免不必要的同步
        if (instance == null) {
            synchronized (DCLSingleton.class) {
                // 第二次检查：确保只有一个实例被创建
                if (instance == null) {
                    instance = new DCLSingleton();
                }
            }
        }
        return instance;
    }
}

为什么需要volatile？

  • 保证可见性：一个线程修改后，其他线程立即可见
  • 禁止指令重排序：防止 instance = new DCLSingleton() 被重排序
  • new操作分为三步：分配内存、初始化对象、引用指向内存地址

静态内部类（Static Inner Class） - 最推荐的实现方式之一

Java
public class StaticInnerSingleton {
    private StaticInnerSingleton() {}

    // 静态内部类持有外部类的实例
    private static class Holder {
        private static final StaticInnerSingleton INSTANCE = new StaticInnerSingleton();
    }

    public static StaticInnerSingleton getInstance() {
        return Holder.INSTANCE;
    }
}

实现原理：

  • 外部类加载时，静态内部类不会立即加载
  • 只有调用getInstance()时，JVM才加载Holder类
  • JVM保证类初始化时的线程安全性
  • 不需要synchronized关键字，性能最优

枚举（Enum） - 最安全的实现方式，Effective Java推荐

Java
public enum EnumSingleton {
    INSTANCE;

    // 可以添加业务方法
    public void doSomething() {
        System.out.println("单例方法执行");
    }
}

// 使用
EnumSingleton instance = EnumSingleton.INSTANCE;
instance.doSomething();

枚举单例的优势：

  • 线程安全：枚举类型由JVM保证线程安全
  • 防止反射攻击：枚举构造器私有且无法通过反射调用
  • 防止序列化破坏：枚举自带序列化机制，readResolve自动处理
  • 代码简洁，实现简单

优缺点分析

优点

  • 全局唯一：确保只有一个实例存在，避免资源重复创建
  • 全局访问：提供统一的访问点，方便使用
  • 延迟加载：部分实现方式支持懒加载，节省资源
  • 线程安全：多种实现方式保证多线程环境下的安全性

缺点

  • 违反单一职责：单例类既要管理实例又要执行业务逻辑
  • 隐藏依赖：通过全局访问点使用，依赖关系不明显
  • 测试困难：全局状态可能影响单元测试的隔离性
  • 扩展困难：继承和扩展相对复杂
