ArrayList 详解

底层实现：动态数组

核心特点：

  • 随机访问 O(1)，插入删除 O(n)
  • 默认初始容量 10，扩容为原来的 1.5 倍
  • 非线程安全
JAVA
// 指定初始容量，避免频繁扩容
List<Integer> list = new ArrayList<>(100);

// 批量添加
list.addAll(Arrays.asList(1, 2, 3, 4, 5));

// 四种遍历方式
// 1. for-each
for (Integer i : list) {
    System.out.println(i);
}

// 2. 迭代器
Iterator<Integer> it = list.iterator();
while (it.hasNext()) {
    System.out.println(it.next());
}

// 3. 索引遍历（适合随机访问）
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}

// 4. forEach + Lambda
list.forEach(System.out::println);
⚠️ 注意事项：遍历时删除元素需要使用迭代器的 remove() 方法，或使用 removeIf()，直接使用集合的 remove() 会导致 ConcurrentModificationException。
🔗

LinkedList 详解

底层实现：双向链表

核心特点：

  • 插入删除 O(1)，随机访问 O(n)
  • 内存开销比 ArrayList 大（需存储前后指针）
  • 可作为队列、栈、双端队列使用
JAVA
LinkedList<Integer> list = new LinkedList<>();

// 作为队列使用（FIFO）
list.offer(1);      // 入队
list.poll();        // 出队
list.peek();        // 查看队首

// 作为栈使用（LIFO）
list.push(1);       // 入栈
list.pop();         // 出栈

// 作为双端队列使用
list.addFirst(1);
list.addLast(2);
list.removeFirst();
list.removeLast();
🗺️

HashMap 详解

底层实现：哈希表（数组 + 链表/红黑树）

核心特点：

  • 基于哈希函数实现 O(1) 时间复杂度的增删改查
  • JDK 8 后，链表长度超过 8 时转为红黑树
  • 非线程安全，允许 null 键和 null 值
📚 深入学习：HashMap 的详细源码分析请参考 HashMapSource 页面。
JAVA
Map<String, Integer> map = new HashMap<>();

map.put("a", 1);
map.putIfAbsent("b", 2);              // 不存在才放入
map.computeIfAbsent("c", k -> 3);     // 不存在时计算
map.compute("a", (k, v) -> v + 1);    // 更新值
map.merge("a", 1, Integer::sum);      // 合并值

// 遍历
map.forEach((k, v) -> System.out.println(k + "=" + v));
📝

LinkedHashMap 详解

核心特点：继承自 HashMap，使用双向链表维护插入顺序或访问顺序

应用场景：LRU 缓存、需要保持顺序的 Map

JAVA
// 访问顺序模式（accessOrder = true）
LinkedHashMap<String, Integer> map = new LinkedHashMap<>(16, 0.75f, true);
map.put("a", 1);
map.put("b", 2);
map.put("c", 3);

map.get("a"); // 访问后，a 移到最后

// LRU 缓存实现（超过容量删除最老的）
LinkedHashMap<String, Integer> lru = new LinkedHashMap<String, Integer>(16, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, Integer> eldest) {
        return size() > 100; // 超过 100 个删除最老的
    }
};
🌳

TreeMap 详解

底层实现：红黑树

核心特点：

  • 按键的自然顺序或自定义比较器排序
  • 查找、插入、删除都是 O(log n)
  • 支持高效的范围查询
JAVA
TreeMap<Integer, String> map = new TreeMap<>();

map.put(3, "c");
map.put(1, "a");
map.put(2, "b");

// 自动排序后 keySet: [1, 2, 3]

// 范围查询
map.subMap(1, 3);      // [1, 2) - 包含 1，不包含 3
map.headMap(2);        // 小于 2 的键
map.tailMap(2);        // 大于等于 2 的键

// 获取最值
map.firstKey();        // 1
map.lastKey();         // 3
map.lowerKey(2);       // 1（小于 2 的最大 key）
map.higherKey(2);      // 3（大于 2 的最小 key）
🎯

HashSet / TreeSet 详解

底层实现：HashMap（value 固定为一个 Object 占位符）

核心特点：

  • 不允许重复元素
  • 无序存储
  • 可以存储 null
  • contains 操作 O(1)
JAVA
Set<String> set = new HashSet<>();
set.add("a");
set.add("b");
set.contains("a");  // O(1)

// 去重操作
List<Integer> list = Arrays.asList(1, 2, 2, 3, 3, 3);
Set<Integer> unique = new HashSet<>(list);  // [1, 2, 3]

底层实现：TreeMap（红黑树）

核心特点：

  • 元素有序（自然顺序或自定义比较器）
  • 不允许重复元素
  • 不支持 null 元素
JAVA
TreeSet<Integer> set = new TreeSet<>();
set.add(3);
set.add(1);
set.add(2);

// 有序遍历
set.first();           // 1
set.last();            // 3
set.lower(2);          // 1
set.higher(2);         // 3
set.subSet(1, 3);      // [1, 2]
⏫

PriorityQueue 详解

底层实现：最小堆（完全二叉树）

核心特点：

  • 最小元素在队首（默认）
  • 入队出队 O(log n)
  • 不支持 null 元素
JAVA
// 默认最小堆
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(3);
minHeap.offer(1);
minHeap.offer(2);
minHeap.poll(); // 返回 1

// 最大堆（使用反向比较器）
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
maxHeap.offer(3);
maxHeap.offer(1);
maxHeap.offer(2);
maxHeap.poll(); // 返回 3

// Top K 问题：找出数组中最大的 K 个数
int[] arr = {1, 5, 2, 8, 3};
PriorityQueue<Integer> heap = new PriorityQueue<>();
for (int num : arr) {
    heap.offer(num);
    if (heap.size() > 3) {
        heap.poll(); // 移除最小的，保留最大的 3 个
    }
}
🔒

ConcurrentHashMap 详解

线程安全的 HashMap

并发实现演进：

  • JDK 1.7：分段锁（Segment），每个 Segment 是一个独立的哈希表
  • JDK 1.8：CAS + synchronized，锁链表头节点或红黑树根节点
JAVA
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// 原子操作
map.putIfAbsent("a", 1);
map.computeIfAbsent("b", k -> 2);
map.merge("a", 1, Integer::sum);

// 批量操作
map.forEach((k, v) -> System.out.println(k + "=" + v));

// 并发计数
map.reduceValues(1, Integer::sum);
📄

CopyOnWriteArrayList 详解

核心机制：写时复制（Copy-On-Write）

实现原理：

  • 写操作时，复制一份新数组，在新数组上修改
  • 读操作不需要加锁，性能极高
  • 修改完成后，将引用指向新数组

适用场景：读多写少的并发场景，遍历操作远多于修改操作

JAVA
CopyOnWriteArrayList<Integer> list = new CopyOnWriteArrayList<>();
list.add(1);

// 遍历时可以修改，不会抛出 ConcurrentModificationException
for (Integer i : list) {
    list.add(2); // 允许，但遍历时看不到新添加的元素
}
💡 注意：CopyOnWriteArrayList 的写操作开销较大（需要复制整个数组），适合读多写少的场景，不适合频繁写入的场景。

面试高频考点

Q1: ArrayList 和 LinkedList 如何选择？
操作ArrayListLinkedList说明
随机访问 get(i)O(1)O(n)ArrayList 基于数组下标直接访问
头部插入/删除O(n)O(1)ArrayList 需要移动后续所有元素
尾部插入O(1) 均摊O(1)ArrayList 扩容时 O(n)
内存占用连续内存每节点额外两个指针LinkedList 内存开销更大
💡
实际上 LinkedList 由于节点分散、CPU缓存命中率低，大多数场景性能不如 ArrayList。优先选 ArrayList，只在频繁头部操作时考虑 LinkedList（或用 ArrayDeque）。
Q2: HashMap 的扩容机制？为什么加载因子是 0.75？

扩容时机：当 size > capacity × loadFactor（默认 16 × 0.75 = 12）时触发，新容量变为原来的 2 倍。

JDK 8 扩容优化：元素新位置只有两种可能——原位置或原位置+原容量，通过判断 hash & oldCap 是否为 0 决定，无需重新计算 hash。

ℹ️
加载因子 0.75 是空间和时间的折中：太大（如1.0）冲突多，查询退化；太小（如0.5）频繁扩容，空间浪费。0.75 在泊松分布下冲突概率最优（链表长度达到8的概率约千万分之一）。
Q3: ConcurrentHashMap 在 JDK 7 和 JDK 8 的实现有何不同？
特性JDK 7（Segment分段锁）JDK 8（CAS + synchronized）
锁粒度Segment（默认16个）单个桶（Node）
并发度固定为 Segment 数量等于桶的数量（最细粒度）
数据结构Segment[] + HashEntry[]Node[] + 链表/红黑树
size() 方法多次尝试无锁统计LongAdder 分段累加
Q4: HashMap 遍历时删除元素会有什么问题？如何正确操作？

直接在 for-each 中调用 map.remove() 会抛出 ConcurrentModificationException（modCount 被修改，迭代器检测到并发修改）。

Java
Map<String, Integer> map = new HashMap<>();

// ❌ 错误：ConcurrentModificationException
for (String key : map.keySet()) {
    if (someCondition) map.remove(key);
}

// ✅ 方案1：使用迭代器
Iterator<Map.Entry<String, Integer>> it = map.entrySet().iterator();
while (it.hasNext()) {
    Map.Entry<String, Integer> entry = it.next();
    if (someCondition) it.remove(); // 调用迭代器的remove
}

// ✅ 方案2：JDK 8 removeIf（推荐）
map.entrySet().removeIf(e -> someCondition);
⚠️

实战陷阱

⚠️
陷阱1：Arrays.asList() 返回的 List 不支持增删 Arrays.asList() 返回的是固定大小的 List（Arrays$ArrayList），调用 add/remove 会抛出 UnsupportedOperationException。需要真正可变的 List 应使用 new ArrayList<>(Arrays.asList(...))。
⚠️
陷阱2：HashMap 的 key 使用可变对象 如果 key 对象在 put 之后修改了影响 hashCode 的字段，get 时会找不到（hash值变了，定位到不同的桶）。key 应使用不可变对象（String、Integer等）。
⚠️
陷阱3：subList 的修改会影响原 List ArrayList.subList() 返回的是原列表的视图，对 subList 的增删改会直接影响原列表，反之亦然。同时对原列表结构性修改后再操作 subList 会抛出 ConcurrentModificationException。
⚖️
