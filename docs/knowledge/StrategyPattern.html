策略模式

定义一系列算法，把它们封装起来，并使它们可以相互替换。策略模式让算法独立于使用它的客户端而变化，避免大量的if-else条件判断。

📖

模式定义与意图

定义：策略模式（Strategy Pattern）是一种行为型设计模式，它定义了算法族，分别封装起来，让它们之间可以互相替换，此模式让算法的变化独立于使用算法的客户。

核心思想：将不同的算法或行为封装成独立的策略类，上下文类通过组合的方式使用策略，而不是通过继承或大量的if-else判断。

classDiagram class Strategy { <> +execute() } class ConcreteStrategyA class ConcreteStrategyB class Context { -strategy: Strategy +setStrategy(Strategy) +executeStrategy() } Strategy <|.. ConcreteStrategyA Strategy <|.. ConcreteStrategyB Context o-- Strategy

角色说明：

  • Strategy（策略接口）：定义所有支持的算法的公共接口
  • ConcreteStrategy（具体策略）：实现具体的算法
  • Context（上下文）：维护一个Strategy的引用，用于调用具体策略
🎯

适用场景

以下场景适合使用策略模式：

  • 支付方式选择：支付宝、微信支付、银行卡支付等不同支付方式
  • 排序算法选择：快速排序、冒泡排序、归并排序等不同排序策略
  • 打折策略：满减、折扣、会员价等不同的促销策略
  • 路线规划：步行、驾车、公交等不同出行策略
  • 数据加密：MD5、SHA、AES等不同加密算法选择
  • 消息推送：短信、邮件、App推送等不同推送渠道

代码实现

场景：支付策略 - 不同支付方式使用不同策略

Java
// 策略接口
interface PaymentStrategy {
    void pay(double amount);
    String getName();
}

// 具体策略：支付宝
class AlipayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("支付宝支付：" + amount + "元");
    }

    @Override
    public String getName() {
        return "支付宝";
    }
}

// 具体策略：微信
class WechatStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("微信支付：" + amount + "元");
    }

    @Override
    public String getName() {
        return "微信";
    }
}

// 具体策略：银行卡
class BankCardStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("银行卡支付：" + amount + "元");
    }

    @Override
    public String getName() {
        return "银行卡";
    }
}

// 上下文类：支付处理器
class PaymentContext {
    private PaymentStrategy strategy;

    public void setStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }

    public void executePayment(double amount) {
        if (strategy == null) {
            throw new IllegalStateException("未设置支付方式");
        }
        System.out.println("使用" + strategy.getName() + "支付");
        strategy.pay(amount);
    }
}

// 使用
PaymentContext context = new PaymentContext();
context.setStrategy(new AlipayStrategy());
context.executePayment(100.0);

context.setStrategy(new WechatStrategy());
context.executePayment(200.0);

场景：排序策略 - 不同排序算法作为不同策略

Java
// 排序策略接口
interface SortStrategy {
    <T extends Comparable<T>> void sort(T[] arr);
    String getName();
}

// 快速排序策略
class QuickSortStrategy implements SortStrategy {
    @Override
    public <T extends Comparable<T>> void sort(T[] arr) {
        quickSort(arr, 0, arr.length - 1);
    }

    private <T extends Comparable<T>> void quickSort(T[] arr, int low, int high) {
        if (low < high) {
            int pivot = partition(arr, low, high);
            quickSort(arr, low, pivot - 1);
            quickSort(arr, pivot + 1, high);
        }
    }

    private <T extends Comparable<T>> int partition(T[] arr, int low, int high) {
        T pivot = arr[high];
        int i = low - 1;
        for (int j = low; j < high; j++) {
            if (arr[j].compareTo(pivot) <= 0) {
                i++;
                T temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
            }
        }
        T temp = arr[i + 1];
        arr[i + 1] = arr[high];
        arr[high] = temp;
        return i + 1;
    }

    @Override
    public String getName() {
        return "快速排序";
    }
}

// 冒泡排序策略
class BubbleSortStrategy implements SortStrategy {
    @Override
    public <T extends Comparable<T>> void sort(T[] arr) {
        for (int i = 0; i < arr.length - 1; i++) {
            for (int j = 0; j < arr.length - 1 - i; j++) {
                if (arr[j].compareTo(arr[j + 1]) > 0) {
                    T temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                }
            }
        }
    }

    @Override
    public String getName() {
        return "冒泡排序";
    }
}

// 使用
Integer[] arr = {5, 2, 8, 1, 9};
SortStrategy sortStrategy = new QuickSortStrategy();
sortStrategy.sort(arr);
System.out.println("排序结果：" + Arrays.toString(arr));

优化：策略 + 工厂模式 - 使用工厂获取策略，避免直接创建策略对象

Java
// 策略工厂
class PaymentStrategyFactory {
    private static final Map<String, PaymentStrategy> STRATEGIES = new HashMap<>();

    static {
        STRATEGIES.put("alipay", new AlipayStrategy());
        STRATEGIES.put("wechat", new WechatStrategy());
        STRATEGIES.put("bank", new BankCardStrategy());
    }

    public static PaymentStrategy getStrategy(String type) {
        PaymentStrategy strategy = STRATEGIES.get(type);
        if (strategy == null) {
            throw new IllegalArgumentException("不支持的支付方式：" + type);
        }
        return strategy;
    }
}

// 使用枚举简化策略（避免创建过多策略类）
enum PaymentType implements PaymentStrategy {
    ALIPAY {
        @Override
        public void pay(double amount) {
            System.out.println("支付宝支付：" + amount + "元");
        }
        @Override
        public String getName() { return "支付宝"; }
    },
    WECHAT {
        @Override
        public void pay(double amount) {
            System.out.println("微信支付：" + amount + "元");
        }
        @Override
        public String getName() { return "微信"; }
    },
    BANK_CARD {
        @Override
        public void pay(double amount) {
            System.out.println("银行卡支付：" + amount + "元");
        }
        @Override
        public String getName() { return "银行卡"; }
    }
}

// 使用
PaymentStrategy strategy = PaymentStrategyFactory.getStrategy("wechat");
strategy.pay(200.0);

// 使用枚举策略
PaymentType.ALIPAY.pay(300.0);

Spring中的策略模式 - 利用Spring IoC容器管理策略

Java
// 策略接口
public interface PaymentStrategy {
    void pay(double amount);
    String getType();
}

// 具体策略
@Component
public class AlipayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("支付宝支付：" + amount);
    }

    @Override
    public String getType() {
        return "alipay";
    }
}

@Component
public class WechatStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("微信支付：" + amount);
    }

    @Override
    public String getType() {
        return "wechat";
    }
}

// 服务类：自动注入所有策略
@Service
public class PaymentService {
    // Spring自动注入所有PaymentStrategy实现
    private final Map<String, PaymentStrategy> strategyMap;

    public PaymentService(List<PaymentStrategy> strategies) {
        this.strategyMap = strategies.stream()
            .collect(Collectors.toMap(
                PaymentStrategy::getType,
                strategy -> strategy
            ));
    }

    public void pay(String type, double amount) {
        PaymentStrategy strategy = strategyMap.get(type);
        if (strategy == null) {
            throw new IllegalArgumentException("不支持的支付方式");
        }
        strategy.pay(amount);
    }
}

优缺点分析

优点

  • 开闭原则：新增算法只需新增策略类，无需修改原有代码
  • 避免if-else：消除复杂的条件判断语句，代码更清晰
  • 算法复用：策略可被多个上下文复用
  • 算法独立：算法变化不影响使用算法的客户端
  • 运行时切换：可以在运行时动态切换策略

缺点

  • 客户端需了解策略：客户端必须知道所有策略类，选择合适的策略
  • 类数量增加：每个策略对应一个类，增加系统复杂度
  • 通信开销：策略与上下文之间可能存在数据传递开销
  • 过度设计：策略较少时，使用简单if-else可能更直观

与其他模式的关系

相关模式 关系说明
工厂模式 工厂模式用于创建策略对象，策略模式用于切换算法
状态模式 状态模式是策略模式的变体，策略由状态决定而非客户端
桥接模式 两者都使用组合替代继承，但关注点不同
模板方法 模板方法在基类中定义算法骨架，策略模式将整个算法封装

策略模式 vs 状态模式：两者结构相似，但意图不同。策略模式由客户端选择策略，状态模式的状态转换由上下文自动管理。
