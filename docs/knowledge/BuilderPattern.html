🏷️ BUILDER · 创建型模式

建造者模式

将复杂对象的构建与表示分离，使同样的构建过程可创建不同表示。链式 API 设计的核心思想，Lombok @Builder、StringBuilder、OkHttp Request.Builder 均是其应用。

📖

为什么需要 Builder？

当一个类有很多参数（尤其是可选参数）时，传统方案有明显问题：

方案问题
重叠构造器（Telescoping Constructor）多个构造器难以阅读，参数顺序容易弄错，扩展困难
JavaBean（setter）对象在多次 setter 调用间处于不一致状态，无法创建不可变对象
Builder 模式✅ 链式调用清晰，支持不可变对象，参数有默认值，build() 时校验
Java - 三种方案对比
// ❌ 方案1：重叠构造器 - 难以阅读，哪个参数是哪个？
User user1 = new User("张三", 25, "zhangsan@email.com", "13800138000", "北京");

// ❌ 方案2：JavaBean - 对象临时不一致，无法不可变
User user2 = new User();
user2.setName("张三");
user2.setAge(25);
// 此时对象处于"半构建"状态，其他线程可能访问到不完整对象

// ✅ 方案3：Builder - 清晰、安全、支持不可变
User user3 = User.builder()
    .name("张三")
    .age(25)
    .email("zhangsan@email.com")
    .phone("13800138000")
    .address("北京")
    .build(); // 一次性构建完整对象

手写链式 Builder

Java
public class Computer {
    // 必需属性（final，不可变）
    private final String cpu;
    private final String ram;

    // 可选属性
    private final String storage;
    private final String gpu;
    private final String monitor;
    private final boolean wifi;

    // 私有构造器，只能通过 Builder 创建
    private Computer(Builder builder) {
        this.cpu = builder.cpu;
        this.ram = builder.ram;
        this.storage = builder.storage;
        this.gpu = builder.gpu;
        this.monitor = builder.monitor;
        this.wifi = builder.wifi;
    }

    // 静态 Builder 类
    public static class Builder {
        // 必需属性
        private final String cpu;
        private final String ram;

        // 可选属性，预设默认值
        private String storage = "256GB SSD";
        private String gpu = null;
        private String monitor = "15寸";
        private boolean wifi = true;

        // Builder 构造器只包含必需参数
        public Builder(String cpu, String ram) {
            if (cpu == null || ram == null) {
                throw new IllegalArgumentException("CPU 和 RAM 为必需参数");
            }
            this.cpu = cpu;
            this.ram = ram;
        }

        // 链式方法：每个方法返回 this（Builder 本身）
        public Builder storage(String storage) { this.storage = storage; return this; }
        public Builder gpu(String gpu) { this.gpu = gpu; return this; }
        public Builder monitor(String monitor) { this.monitor = monitor; return this; }
        public Builder wifi(boolean wifi) { this.wifi = wifi; return this; }

        // build() 方法：创建最终对象，并可进行参数校验
        public Computer build() {
            // 参数校验
            if (storage == null || storage.isEmpty()) {
                throw new IllegalStateException("存储不能为空");
            }
            return new Computer(this);
        }
    }

    @Override
    public String toString() {
        return "Computer{cpu='" + cpu + "', ram='" + ram + "', storage='" + storage +
               "', gpu='" + gpu + "', monitor='" + monitor + "', wifi=" + wifi + "}";
    }
}

// 使用：链式调用，清晰易读
Computer computer = new Computer.Builder("Intel i9", "32GB")
    .storage("1TB NVMe SSD")
    .gpu("RTX 4090")
    .monitor("32寸 4K")
    .wifi(true)
    .build();

// 最简版（仅必需参数，其他用默认值）
Computer basicPC = new Computer.Builder("Intel i5", "8GB").build();

Lombok @Builder 完整用法

Java - Lombok @Builder
// 基础用法：Lombok 自动生成 Builder
@Data
@Builder
public class UserDTO {
    private String name;
    private Integer age;
    private String email;

    // @Builder.Default：指定字段默认值
    @Builder.Default
    private String role = "USER";

    @Builder.Default
    private LocalDateTime createdAt = LocalDateTime.now();

    // @Singular：生成 add 单个元素的方法
    @Singular("tag")
    private List<String> tags;
}

// 使用
UserDTO user = UserDTO.builder()
    .name("张三")
    .age(25)
    .email("zhang@example.com")
    // role 和 createdAt 使用默认值
    .tag("Java")      // @Singular 生成单个添加方法
    .tag("Spring")
    .build();

// 注意：@Builder 与 @Data 组合时，需要手动添加 @AllArgsConstructor @NoArgsConstructor
// 否则 JPA 实体无法正常使用（JPA 需要无参构造器）
@Entity
@Data
@Builder
@NoArgsConstructor   // ← JPA 需要
@AllArgsConstructor  // ← @Builder 需要
public class User {
    @Id
    private Long id;
    private String name;
}

// toBuilder()：基于现有对象创建修改版（需加 toBuilder=true）
@Builder(toBuilder = true)
public class Config { ... }

Config newConfig = existingConfig.toBuilder()
    .timeout(30)   // 只修改需要的字段
    .build();
⚠️ 常见陷阱：@Builder 会生成全参构造器，导致 JPA 实体缺少无参构造器。必须同时加 @NoArgsConstructor 和 @AllArgsConstructor。

面试高频考点

Q1：Builder 模式 vs 构造器重载 vs JavaBean 如何选择？
  • 构造器重载：参数少（≤4个），都是必需参数时可用；参数多时可读性差
  • JavaBean：参数多且都可选时；但无法创建不可变对象，有线程安全隐患
  • Builder：推荐。参数多（尤其有可选参数）、需要不可变对象、需要参数校验时使用
规则：如果类的构造器或静态工厂的参数超过 4 个，就应该考虑 Builder 模式（Effective Java）。
Q2：Builder 为什么更适合不可变对象？
不可变对象（所有字段 final）必须在构造时一次性设置所有字段。多参数构造器可读性差，JavaBean 的 setter 无法用于 final 字段。Builder 提供了流式的构建过程，最后调用 build() 一次性创建 final 字段的对象，兼顾了可读性和不可变性。不可变对象天然线程安全（无需同步）。
Q3：框架中哪些地方用了 Builder 模式？
  • StringBuilder / StringBuffer：Java 基础库，链式 append 构建字符串
  • MyBatis SqlSessionFactoryBuilder：构建 SqlSessionFactory
  • Spring RestTemplate / WebClient：WebClient.builder().baseUrl().build()
  • OkHttp：Request.Builder().url().header().build()
  • Retrofit：new Retrofit.Builder().baseUrl().addConverterFactory().build()
  • Spring Security：HttpSecurity 链式配置就是 Builder 模式
Q4：传统 Builder（Director）与现代链式 Builder 的区别？
  • 传统 GoF Builder：有 Director 类控制构建顺序，Builder 接口定义步骤，适合构建流程固定但结果不同的场景（如不同风格的房子）
  • 现代链式 Builder（Effective Java 推荐）：静态内部 Builder 类，链式方法返回 this，更适合"可选参数多"的对象创建，代码更简洁
  • 实际工作：现代链式 Builder 更常用，Lombok @Builder 就是基于此

优缺点分析

✅ 优点

  • 链式调用，可读性强
  • 参数有默认值，可选参数灵活
  • 支持创建不可变对象（线程安全）
  • build() 方法可进行参数完整性校验
  • 分离构建和表示，同一过程可创建不同对象

❌ 缺点

  • 代码量增多（需要写 Builder 类）
  • 有一定性能开销（创建 Builder 对象）
  • Lombok @Builder 与 JPA 存在兼容问题
🏛️

传统 GoF Builder（带 Director）

GoF 原始定义中包含 Director（指挥者），负责控制构建流程。适合"构建步骤固定但结果不同"的场景（如报告生成器）。

Java - 传统 Builder + Director
// 产品：电脑
class Computer {
    private String cpu, ram, storage, os;
    // setters & toString...
}

// Builder 接口：定义构建步骤
interface ComputerBuilder {
    void buildCPU(String cpu);
    void buildRAM(String ram);
    void buildStorage(String storage);
    void buildOS(String os);
    Computer getResult();
}

// 具体 Builder
class GamingComputerBuilder implements ComputerBuilder {
    private Computer computer = new Computer();
    @Override public void buildCPU(String cpu) { computer.setCpu(cpu); }
    @Override public void buildRAM(String ram) { computer.setRam(ram); }
    @Override public void buildStorage(String storage) { computer.setStorage(storage); }
    @Override public void buildOS(String os) { computer.setOs(os); }
    @Override public Computer getResult() { return computer; }
}

// Director：封装固定的构建流程
class ComputerDirector {
    private ComputerBuilder builder;
    public ComputerDirector(ComputerBuilder builder) { this.builder = builder; }

    // 构建游戏电脑（固定流程）
    public Computer buildGamingPC() {
        builder.buildCPU("Intel i9-13900K");
        builder.buildRAM("64GB DDR5");
        builder.buildStorage("2TB NVMe SSD");
        builder.buildOS("Windows 11");
        return builder.getResult();
    }

    // 构建办公电脑（不同配置，相同步骤）
    public Computer buildOfficePC() {
        builder.buildCPU("Intel i5-13500");
        builder.buildRAM("16GB DDR4");
        builder.buildStorage("512GB SSD");
        builder.buildOS("Windows 11");
        return builder.getResult();
    }
}

// 使用
ComputerDirector director = new ComputerDirector(new GamingComputerBuilder());
Computer gamingPC = director.buildGamingPC();
对比维度传统 GoF Builder现代链式 Builder
Director有，负责控制构建顺序无，客户端直接调用链式方法
Builder 接口通常有多个步骤方法每个字段一个方法，返回 this
适用场景构建流程固定、产品复杂可选参数多、需要不可变对象
常见实现SQL 查询构建、报告生成Lombok @Builder、OkHttp Request

实战：统一响应体 Builder

项目中最常用的 Builder 应用：封装统一 API 响应格式，避免直接构造对象。

Java
// 统一响应体（不可变）
public class Result {
    private final int code;
    private final String message;
    private final T data;
    private final long timestamp;

    private Result(Builder builder) {
        this.code = builder.code;
        this.message = builder.message;
        this.data = builder.data;
        this.timestamp = System.currentTimeMillis();
    }

    // 静态工厂方法 + Builder，提供便捷入口
    public static  Result success(T data) {
        return new Builder().code(200).message("success").data(data).build();
    }

    public static  Result fail(int code, String message) {
        return new Builder().code(code).message(message).build();
    }

    public static  Builder builder() { return new Builder<>(); }

    public static class Builder {
        private int code = 200;
        private String message = "success";
        private T data;

        public Builder code(int code) { this.code = code; return this; }
        public Builder message(String msg) { this.message = msg; return this; }
        public Builder data(T data) { this.data = data; return this; }

        public Result build() {
            if (code < 100 || code > 599) {
                throw new IllegalArgumentException("无效的 HTTP 状态码：" + code);
            }
            return new Result<>(this);
        }
    }
}

// 使用示例
@GetMapping("/users/{id}")
public Result getUser(@PathVariable Long id) {
    UserDTO user = userService.findById(id);
    return Result.success(user);
    // 等同于：Result.builder().code(200).message("success").data(user).build()
}

@ExceptionHandler(NotFoundException.class)
public Result handleNotFound(NotFoundException e) {
    return Result.fail(404, e.getMessage());
}

相关模式关系说明
