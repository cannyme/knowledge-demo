📚 DESIGN PATTERN

建造者模式

将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。将复杂对象的创建过程抽象出来，使这个抽象过程的不同实现方法可以构造出不同表现（属性）的对象。

🎯

模式定义与意图

建造者模式（Builder Pattern）是一种创建型设计模式，用于创建复杂对象。它将对象的构建过程与其表示分离，使得相同的构建过程可以创建不同的表示。

核心思想：将复杂对象的构造逻辑从业务逻辑中分离，通过一步步（step-by-step）的方式构建复杂对象，而不是一次性传入所有参数。

生活中的类比：

  • 汽车组装工厂：按照相同流程组装不同配置的汽车
  • 麦当劳套餐：选择主食、饮料、配餐，组合成完整套餐
  • 乐高积木：使用相同模块搭建不同造型的建筑
🏗️

角色说明：

角色说明
Product产品角色，需要构建的复杂对象
Builder抽象建造者，定义创建产品各个部分的抽象接口
ConcreteBuilder具体建造者，实现Builder接口，构建和装配各个部件
Director指挥者，隔离客户与创建过程，控制构建步骤
Client客户端，使用Director或直接使用Builder创建产品
📝

四种实现方式

经典Builder模式（带Director）

完整实现包含Director角色，由Director控制构建流程。

Java
// 产品
class House {
    private String foundation;
    private String structure;
    private String roof;
    private String interior;

    public void setFoundation(String f) { this.foundation = f; }
    public void setStructure(String s) { this.structure = s; }
    public void setRoof(String r) { this.roof = r; }
    public void setInterior(String i) { this.interior = i; }

    @Override
    public String toString() {
        return String.format("House{%s, %s, %s, %s}",
            foundation, structure, roof, interior);
    }
}

// 建造者接口
interface HouseBuilder {
    void buildFoundation();
    void buildStructure();
    void buildRoof();
    void buildInterior();
    House getHouse();
}

// 具体建造者
class WoodenHouseBuilder implements HouseBuilder {
    private House house = new House();

    @Override public void buildFoundation() {
        house.setFoundation("木质地基");
    }
    @Override public void buildStructure() {
        house.setStructure("木质结构");
    }
    @Override public void buildRoof() {
        house.setRoof("木质屋顶");
    }
    @Override public void buildInterior() {
        house.setInterior("木质内饰");
    }
    @Override public House getHouse() { return house; }
}

// 指挥者
class HouseDirector {
    public House construct(HouseBuilder builder) {
        builder.buildFoundation();
        builder.buildStructure();
        builder.buildRoof();
        builder.buildInterior();
        return builder.getHouse();
    }
}

// 使用
HouseDirector director = new HouseDirector();
House woodenHouse = director.construct(new WoodenHouseBuilder());

链式调用（流畅接口 Fluent Interface）

最常用的实现方式，支持方法链式调用，可读性极强。

Java
class Computer {
    // 必需属性
    private final String cpu;
    private final String ram;

    // 可选属性
    private final String storage;
    private final String gpu;
    private final boolean wifi;

    // 私有构造器
    private Computer(Builder builder) {
        this.cpu = builder.cpu;
        this.ram = builder.ram;
        this.storage = builder.storage;
        this.gpu = builder.gpu;
        this.wifi = builder.wifi;
    }

    // Builder类
    public static class Builder {
        // 必需属性
        private final String cpu;
        private final String ram;

        // 可选属性（带默认值）
        private String storage = "256GB SSD";
        private String gpu = null;
        private boolean wifi = true;

        // 构造器包含必需参数
        public Builder(String cpu, String ram) {
            this.cpu = cpu;
            this.ram = ram;
        }

        // 链式方法返回this
        public Builder storage(String storage) {
            this.storage = storage;
            return this;
        }
        public Builder gpu(String gpu) {
            this.gpu = gpu;
            return this;
        }
        public Builder wifi(boolean wifi) {
            this.wifi = wifi;
            return this;
        }

        // 构建方法
        public Computer build() {
            return new Computer(this);
        }
    }
}

// 使用：优雅的链式调用
Computer computer = new Computer.Builder("Intel i7", "16GB")
    .storage("512GB SSD")
    .gpu("RTX 3080")
    .wifi(true)
    .build();

Lombok @Builder 注解

使用Lombok自动生成Builder代码，最简洁的实现方式。

Java
import lombok.Builder;
import lombok.AllArgsConstructor;
import lombok.AccessLevel;

@Builder
@AllArgsConstructor(access = AccessLevel.PRIVATE)
public class User {
    private String name;
    private Integer age;
    private String email;
    private String phone;
    private String address;
}

// 使用方式
User user = User.builder()
    .name("张三")
    .age(25)
    .email("zhangsan@example.com")
    .phone("13800138000")
    .address("北京市")
    .build();

// Lombok还生成toBuilder()用于修改
User modifiedUser = user.toBuilder()
    .age(26)
    .build();
💡 Lombok优势
自动生成Builder类、build()方法、字段赋值方法，大幅减少样板代码。可通过@Builder.Default设置默认值。

静态内部类方式

将Builder作为产品类的静态内部类，紧密关联，组织清晰。

Java
public class Order {
    private final String orderId;
    private final String customer;
    private final Address address;

    private Order(Builder builder) {
        this.orderId = builder.orderId;
        this.customer = builder.customer;
        this.address = builder.address;
    }

    // 静态内部类Builder
    public static class Builder {
        private String orderId;
        private String customer;
        private Address address;

        public Builder orderId(String orderId) {
            this.orderId = orderId;
            return this;
        }
        public Builder customer(String customer) {
            this.customer = customer;
            return this;
        }
        public Builder address(Address address) {
            this.address = address;
            return this;
        }
        // 嵌套Builder
        public Address.Builder address() {
            return new Address.Builder(this);
        }
        public Order build() {
            return new Order(this);
        }
    }

    // 嵌套对象的Builder
    public static class Address {
        private final String province, city, detail;

        private Address(Builder builder) {
            this.province = builder.province;
            this.city = builder.city;
            this.detail = builder.detail;
        }

        public static class Builder {
            private final Order.Builder orderBuilder;
            private String province, city, detail;

            public Builder(Order.Builder orderBuilder) {
                this.orderBuilder = orderBuilder;
            }
            public Builder province(String p) { this.province = p; return this; }
            public Builder city(String c) { this.city = c; return this; }
            public Builder detail(String d) { this.detail = d; return this; }
            public Order.Builder end() {
                orderBuilder.address(new Address(this));
                return orderBuilder;
            }
        }
    }
}

// 嵌套使用
Order order = new Order.Builder()
    .orderId("ORD-001")
    .customer("张三")
    .address()
        .province("北京市")
        .city("海淀区")
        .detail("中关村大街1号")
        .end()
    .build();
💡

适用场景

  • 参数过多：构造方法参数超过4个，难以记忆和使用
  • 可选参数多：对象有很多可选属性，需要使用大量重载构造方法
  • 不可变对象：需要创建不可变（final字段）对象
  • 复杂构建过程：对象创建需要分步骤进行，有固定顺序
  • 同构不同构：使用相同的构建过程创建不同的产品
⚠️ 与工厂模式的区别
维度建造者模式工厂模式
关注点构建过程，分步骤组装创建结果，直接返回对象
参数多参数、可选参数通常参数较少
控制细粒度控制每一步不关心内部构建细节
产品通常是复杂对象可以是简单或复杂对象

StringBuilder源码分析

StringBuilder是建造者模式的经典应用：

Java
// StringBuilder使用建造者模式
StringBuilder sb = new StringBuilder();
String result = sb.append("Hello")
                  .append(" ")
                  .append("World")
                  .append("!")
                  .insert(0, "Say: ")
                  .toString();  // "Say: Hello World!"

// 核心实现简析
public final class StringBuilder {
    char[] value;  // 存储字符
    int count;     // 实际长度

    // 链式append，返回this
    public StringBuilder append(String str) {
        // 扩容、复制逻辑
        // ...
        return this;  // 关键：返回自身支持链式调用
    }

    public String toString() {
        return new String(value, 0, count);
    }
}
💡 StringBuilder vs StringBuffer
  • StringBuilder：非线程安全，单线程环境下性能更好
  • StringBuffer：线程安全（方法加synchronized），多线程环境下使用
  • 两者都是可变字符串，避免了String的不可变带来的内存开销
⚖️

优缺点分析

✅ 优点
  • 良好的封装性：构建过程与表示分离，客户端无需知道内部细节
  • 更好的可读性：链式调用使代码意图清晰，自描述性强
  • 不可变性支持：轻松创建不可变对象
  • 参数校验：可在build()方法中进行统一校验
  • 避免重叠构造器：不需要编写大量构造方法
❌ 缺点
  • 代码量增加：需要编写Builder类
  • 额外内存开销：创建Builder对象需要额外内存
  • 不适用于简单对象：简单对象直接使用构造方法更清晰
📋

最佳实践

何时使用Builder：当类有多个可选参数，或者构造方法参数超过4个时，考虑使用建造者模式。

使用建议：

  • 必需参数放在Builder构造器中
  • 可选参数使用方法设置，并合理设置默认值
  • 在build()方法中进行参数校验
  • Builder和产品类使用静态内部类关系
  • 考虑使用Lombok减少样板代码
