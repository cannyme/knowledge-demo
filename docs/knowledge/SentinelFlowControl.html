Sentinel流量控制与熔断降级

Sentinel是阿里巴巴开源的流量控制组件，提供流量控制、熔断降级、系统负载保护、热点参数限流等功能，保障微服务的稳定性。

流量控制规则

限流算法

graph LR A[滑动窗口算法] -->|默认| B[统计QPS] C[漏桶算法] -->|平滑| D[匀速处理] E[令牌桶算法] -->|突发| F[允许突发流量]

限流维度

维度说明
QPS每秒请求数限制
线程数并发线程数限制

流控策略

  • 直接拒绝（默认）：QPS超过阈值直接拒绝
  • 关联限流：当关联资源B的QPS超过阈值时，限流资源A（如支付接口繁忙时限制下单接口）
  • 链路限流：只记录从指定入口资源进来的流量

流控效果

  • 快速失败：直接抛出FlowException
  • Warm Up（预热）：冷启动，阈值从threshold/3逐渐增加到threshold
  • 排队等待：请求在队列中排队，按照间隔时间依次通过

限流代码示例

Java
@RestController
public class UserController {

    // 方式1：使用@SentinelResource注解
    @SentinelResource(
        value = "getUser",
        blockHandler = "getUserBlockHandler",
        fallback = "getUserFallback"
    )
    @GetMapping("/user/{id}")
    public User getUser(@PathVariable Long id) {
        return userService.getById(id);
    }

    // 限流处理方法（必须与原方法签名一致，最后加BlockException参数）
    public User getUserBlockHandler(Long id, BlockException ex) {
        return User.builder()
            .id(id)
            .name("限流用户")
            .build();
    }

    // 降级处理方法（处理业务异常）
    public User getUserFallback(Long id, Throwable ex) {
        log.error("查询用户异常", ex);
        return User.builder()
            .id(id)
            .name("降级用户")
            .build();
    }

    // 方式2：代码定义规则
    @PostConstruct
    public void initFlowRules() {
        List<FlowRule> rules = new ArrayList<>();

        FlowRule rule = new FlowRule();
        rule.setResource("getUser");              // 资源名
        rule.setGrade(RuleConstant.FLOW_GRADE_QPS);  // QPS模式
        rule.setCount(100);                       // 每秒100次
        rule.setStrategy(RuleConstant.STRATEGY_DIRECT);
        rule.setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT);

        rules.add(rule);
        FlowRuleManager.loadRules(rules);
    }
}

熔断状态机

熔断策略

策略说明
慢调用比例响应时间超过阈值的调用为慢调用，慢调用比例超过阈值时触发熔断
异常比例异常数/总请求数 比例超过阈值时触发熔断
异常数异常数超过阈值时触发熔断

熔断代码示例

Java
private void initDegradeRules() {
    List<DegradeRule> rules = new ArrayList<>();

    DegradeRule rule = new DegradeRule();
    rule.setResource("callRemoteService");
    rule.setGrade(CircuitBreakerStrategy.SLOW_REQUEST_RATIO.getType());
    rule.setCount(500);                // 慢调用阈值：500ms
    rule.setSlowRatioThreshold(0.5);   // 慢调用比例：50%
    rule.setMinRequestAmount(10);      // 最小请求数：10
    rule.setStatIntervalMs(10000);     // 统计时长：10秒
    rule.setTimeWindow(30);            // 熔断时长：30秒

    rules.add(rule);
    DegradeRuleManager.loadRules(rules);
}

系统保护规则

从系统整体维度（CPU、内存、QPS、RT）进行保护，避免系统过载导致整体不可用。

规则类型说明
Load系统负载（Linux 1分钟平均负载）
CPU使用率CPU使用率超过阈值触发限流
平均RT所有入口流量的平均响应时间
并发线程数并发线程数超过阈值触发限流
入口QPS所有入口流量的QPS
Java
private void initSystemRules() {
    List<SystemRule> rules = new ArrayList<>();

    SystemRule rule = new SystemRule();
    rule.setHighestSystemLoad(10);     // 系统负载阈值
    rule.setHighestCpuUsage(0.8);      // CPU使用率80%
    rule.setAvgRt(500);                // 平均响应时间500ms
    rule.setMaxThread(500);            // 最大并发线程
    rule.setQps(10000);                // 入口QPS阈值

    rules.add(rule);
    SystemRuleManager.loadRules(rules);
}

热点参数限流

针对请求中的热点参数进行限流，如热门商品ID、特定用户ID等。

Java
@GetMapping("/product/{id}")
@SentinelResource(value = "getProduct", blockHandler = "getProductBlock")
public Product getProduct(
    @PathVariable @SentinelParam("productId") String productId) {
    return productService.getById(productId);
}

// 配置热点参数限流规则
private void initParamFlowRules() {
    ParamFlowRule rule = new ParamFlowRule();
    rule.setResource("getProduct");
    rule.setCount(100);                    // 默认阈值100 QPS
    rule.setGrade(RuleConstant.FLOW_GRADE_QPS);
    rule.setDurationInSec(1);              // 统计窗口1秒

    // 特定参数值单独设置阈值
    List<ParamFlowItem> items = new ArrayList<>();

    // 热门商品ID限流阈值更高
    ParamFlowItem item1 = new ParamFlowItem();
    item1.setObject("1001");               // 商品ID
    item1.setClassType(String.class.getName());
    item1.setCount(500);                   // 阈值500
    items.add(item1);

    rule.setParamFlowItemList(items);
    ParamFlowRuleManager.loadRules(Collections.singletonList(rule));
}

授权规则（黑白名单）

根据调用来源限制资源的访问，实现黑白名单控制。

Java
// 实现RequestOriginParser接口
@Component
public class RequestOriginParserDefinition implements RequestOriginParser {

    @Override
    public String parseOrigin(HttpServletRequest request) {
        // 从请求头获取来源标识
        String origin = request.getHeader("S-Origin");
        if (StringUtils.isEmpty(origin)) {
            origin = request.getParameter("origin");
        }
        return origin;
    }
}

// 配置授权规则（只允许app1和app2访问）
private void initAuthorityRules() {
    AuthorityRule rule = new AuthorityRule();
    rule.setResource("getUser");
    rule.setStrategy(RuleConstant.AUTHORITY_WHITE);  // 白名单
    rule.setLimitApp("app1,app2");                   // 只允许app1和app2

    AuthorityRuleManager.loadRules(Collections.singletonList(rule));
}

规则持久化（Nacos）

默认规则存储在内存中，应用重启后丢失。通过Nacos实现规则持久化：

YAML
spring:
  cloud:
    sentinel:
      datasource:
        # 限流规则
        flow:
          nacos:
            server-addr: localhost:8848
            data-id: ${spring.application.name}-flow-rules
            group-id: SENTINEL_GROUP
            rule-type: flow
        # 熔断规则
        degrade:
          nacos:
            server-addr: localhost:8848
            data-id: ${spring.application.name}-degrade-rules
            group-id: SENTINEL_GROUP
            rule-type: degrade
        # 热点参数规则
        param-flow:
          nacos:
            server-addr: localhost:8848
            data-id: ${spring.application.name}-param-rules
            group-id: SENTINEL_GROUP
            rule-type: param-flow

# 访问地址：http://localhost:8080
# 默认账号：sentinel / sentinel

应用接入控制台

YAML
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8080    # 控制台地址
        port: 8719                    # 与控制台通信端口
      eager: true                     # 应用启动立即连接

  • 机器管理：查看和管理应用实例
  • 簇点链路：查看资源调用链路

最佳实践

限流阈值设置

  • 根据压测结果设置，留20%-30%余量
  • 分级限流：网关层全局限流、应用层接口限流、方法层核心方法限流
  • 限流响应返回友好提示和HTTP 429状态码

熔断阈值设置

  • 慢调用RT阈值：根据SLA设置
  • 比例阈值：50%-70%
  • 熔断时长：30秒-60秒

降级策略

  • 返回默认值：适用于查询类接口
  • 返回缓存数据：适用于数据一致性要求不高场景
  • 调用备用服务：适用于有备用服务场景
  • 返回错误提示：适用于无法降级场景
