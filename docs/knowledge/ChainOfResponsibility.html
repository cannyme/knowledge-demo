责任链模式

将请求沿着处理链依次传递，直到某个处理者处理它为止。解耦请求发送者与接收者，让多个对象都有机会处理请求。Servlet Filter、Spring Interceptor、Netty Pipeline 均是其典型应用。

📖

模式结构与两种变体

变体特点典型场景
纯责任链请求只被一个处理者处理，处理后不再传递审批流程（组长处理了就不到经理）
非纯责任链每个处理者都处理请求并继续传递（全员处理）Servlet Filter、Spring Interceptor

三个核心角色：

  • Handler（抽象处理者）：持有 next 引用，定义 handle() 接口
  • ConcreteHandler（具体处理者）：决定是否处理请求，或传给下一个
  • Client（客户端）：组装责任链，向第一个处理者提交请求

代码实现

纯责任链 — 请假审批：只有一个处理者处理，其余跳过

Java
// 抽象处理者
abstract class Approver {
    protected Approver next;
    protected final String name;

    public Approver(String name) { this.name = name; }

    // 链式设置，支持 leader.setNext(manager).setNext(director)
    public Approver setNext(Approver next) {
        this.next = next;
        return next; // 返回 next，支持链式调用
    }

    public abstract void approve(LeaveRequest request);

    // 传递给下一个处理者
    protected void passToNext(LeaveRequest request) {
        if (next != null) {
            System.out.println(name + " 无权处理，上报给 " + next.name);
            next.approve(request);
        } else {
            System.out.println("已到链尾，无人处理：" + request);
        }
    }
}

// 请假申请
class LeaveRequest {
    private final String employee;
    private final int days;
    public LeaveRequest(String employee, int days) {
        this.employee = employee; this.days = days;
    }
    public int getDays() { return days; }
    public String toString() { return employee + " 请假 " + days + " 天"; }
}

// 组长：可审批 1-3 天
class TeamLeader extends Approver {
    public TeamLeader(String name) { super(name); }
    @Override
    public void approve(LeaveRequest request) {
        if (request.getDays() <= 3) {
            System.out.println("✅ [组长-" + name + "] 批准：" + request);
        } else {
            passToNext(request);
        }
    }
}

// 经理：可审批 4-7 天
class Manager extends Approver {
    public Manager(String name) { super(name); }
    @Override
    public void approve(LeaveRequest request) {
        if (request.getDays() <= 7) {
            System.out.println("✅ [经理-" + name + "] 批准：" + request);
        } else {
            passToNext(request);
        }
    }
}

// 总监：7 天以上
class Director extends Approver {
    public Director(String name) { super(name); }
    @Override
    public void approve(LeaveRequest request) {
        System.out.println("✅ [总监-" + name + "] 批准：" + request);
    }
}

// 组装责任链并使用
Approver leader  = new TeamLeader("张三");
Approver manager = new Manager("李四");
Approver director = new Director("王五");

leader.setNext(manager).setNext(director); // 链式组装

leader.approve(new LeaveRequest("小明", 2));  // 组长批准
leader.approve(new LeaveRequest("小红", 5));  // 经理批准
leader.approve(new LeaveRequest("小刚", 10)); // 总监批准

非纯责任链 — 模拟 Servlet FilterChain：每个 Filter 都处理并继续传递

Java
// 模拟 Servlet Filter 接口
interface Filter {
    void doFilter(HttpRequest request, HttpResponse response, FilterChain chain);
}

// FilterChain：用下标驱动，依次调用每个 Filter
class FilterChain {
    private final List filters = new ArrayList<>();
    private int index = 0;

    public FilterChain addFilter(Filter filter) {
        filters.add(filter);
        return this; // 支持链式 addFilter
    }

    public void doFilter(HttpRequest request, HttpResponse response) {
        if (index < filters.size()) {
            Filter filter = filters.get(index++);
            filter.doFilter(request, response, this); // 传入 chain，Filter 决定是否继续
        }
        // index >= size 时，所有 Filter 执行完毕，到达目标 Servlet
    }
}

// 日志 Filter：前后都处理（围绕式）
class LoggingFilter implements Filter {
    @Override
    public void doFilter(HttpRequest req, HttpResponse resp, FilterChain chain) {
        System.out.println("[日志] 请求进入：" + req.getPath());
        long start = System.currentTimeMillis();
        chain.doFilter(req, resp); // 继续传递
        System.out.println("[日志] 请求完成，耗时：" + (System.currentTimeMillis() - start) + "ms");
    }
}

// 认证 Filter：验证失败则短路（不调用 chain.doFilter）
class AuthFilter implements Filter {
    @Override
    public void doFilter(HttpRequest req, HttpResponse resp, FilterChain chain) {
        String token = req.getHeader("Authorization");
        if (token == null || !token.startsWith("Bearer ")) {
            resp.setStatus(401);
            resp.write("Unauthorized"); // 短路，不再传递
            return;
        }
        System.out.println("[认证] Token 验证通过");
        chain.doFilter(req, resp); // 认证通过，继续传递
    }
}

// 限流 Filter
class RateLimitFilter implements Filter {
    private final int maxQps;
    private final AtomicInteger counter = new AtomicInteger(0);

    public RateLimitFilter(int maxQps) { this.maxQps = maxQps; }

    @Override
    public void doFilter(HttpRequest req, HttpResponse resp, FilterChain chain) {
        if (counter.incrementAndGet() > maxQps) {
            resp.setStatus(429);
            resp.write("Too Many Requests");
            return; // 限流，短路
        }
        chain.doFilter(req, resp);
    }
}

// 组装并使用
FilterChain chain = new FilterChain()
    .addFilter(new LoggingFilter())
    .addFilter(new RateLimitFilter(100))
    .addFilter(new AuthFilter());

chain.doFilter(request, response);
关键设计：chain.doFilter() 是否被调用决定请求是否继续传递。Filter 在调用前后都可以执行逻辑（围绕式增强），这也是 Filter 比 Interceptor 功能更强的原因。

现代中间件链写法 — 函数式接口 + Lambda，消灭冗余类

Java
// 函数式中间件接口
@FunctionalInterface
interface Middleware {
    void execute(Request request, Runnable next);
}

// 中间件链（职责链的函数式实现）
class MiddlewareChain {
    private final List middlewares = new ArrayList<>();

    public MiddlewareChain use(Middleware middleware) {
        middlewares.add(middleware);
        return this;
    }

    public void execute(Request request) {
        // 将 List 转为嵌套调用链（从后往前组合）
        Runnable chain = () -> {}; // 末尾：空操作（到达目标处理器）
        for (int i = middlewares.size() - 1; i >= 0; i--) {
            final Middleware middleware = middlewares.get(i);
            final Runnable next = chain;
            chain = () -> middleware.execute(request, next);
        }
        chain.run();
    }
}

// 使用 Lambda 定义中间件，无需创建类
MiddlewareChain chain = new MiddlewareChain()
    .use((req, next) -> {
        System.out.println("[日志] 开始：" + req.getPath());
        next.run(); // 继续链
        System.out.println("[日志] 结束");
    })
    .use((req, next) -> {
        if (req.getHeader("token") == null) {
            System.out.println("[认证] 拒绝：无 token");
            return; // 短路，不调用 next
        }
        next.run();
    })
    .use((req, next) -> {
        System.out.println("[业务] 处理请求：" + req.getPath());
        // 最后一个中间件通常不调用 next（已到达终点）
    });

chain.execute(new Request("/api/users", "Bearer xxx"));

Spring Security FilterChain — 最典型的非纯责任链

Java - Spring Security 过滤器链
// Spring Security 内置了 15+ 个过滤器，按顺序组成责任链
// 请求经过每个 Filter 处理（非纯责任链，全员参与）

// 关键过滤器执行顺序（简化）：
// SecurityContextPersistenceFilter  → 从 Session 加载 SecurityContext
// UsernamePasswordAuthenticationFilter → 处理表单登录
// BasicAuthenticationFilter         → 处理 Basic 认证
// ExceptionTranslationFilter        → 处理认证/授权异常
// FilterSecurityInterceptor         → 做最终权限决策

// 自定义 JWT 过滤器，插入责任链
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    @Autowired
    private JwtTokenProvider tokenProvider;
    @Autowired
    private UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws IOException, ServletException {
        try {
            String token = extractToken(request);
            if (token != null && tokenProvider.validateToken(token)) {
                String username = tokenProvider.getUsernameFromToken(token);
                UserDetails userDetails = userDetailsService.loadUserByUsername(username);
                // 将认证信息放入 SecurityContext
                UsernamePasswordAuthenticationToken auth =
                    new UsernamePasswordAuthenticationToken(userDetails, null,
                        userDetails.getAuthorities());
                SecurityContextHolder.getContext().setAuthentication(auth);
            }
        } catch (Exception e) {
            SecurityContextHolder.clearContext();
        }
        filterChain.doFilter(request, response); // 继续传递，非纯责任链
    }

    private String extractToken(HttpServletRequest request) {
        String header = request.getHeader("Authorization");
        if (header != null && header.startsWith("Bearer ")) {
            return header.substring(7);
        }
        return null;
    }
}

// 注册到 Spring Security 过滤器链
@Configuration
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    @Autowired
    private JwtAuthenticationFilter jwtFilter;

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);
    }
}

Netty ChannelPipeline — 双向责任链

Java - Netty Pipeline
// Netty 的 ChannelPipeline 是责任链的双向变体
// Inbound（入站）：读数据时从头到尾传递
// Outbound（出站）：写数据时从尾到头传递

ServerBootstrap bootstrap = new ServerBootstrap();
bootstrap.childHandler(new ChannelInitializer() {
    @Override
    protected void initChannel(SocketChannel ch) {
        ChannelPipeline pipeline = ch.pipeline();

        // 添加处理器（顺序即责任链顺序）
        pipeline.addLast("decoder",  new HttpRequestDecoder());    // 入站：解码 HTTP
        pipeline.addLast("encoder",  new HttpResponseEncoder());   // 出站：编码 HTTP
        pipeline.addLast("aggregator", new HttpObjectAggregator(65536)); // 入站：聚合
        pipeline.addLast("logging",  new LoggingHandler());        // 双向：日志
        pipeline.addLast("business", new BusinessHandler());       // 入站：业务逻辑
    }
});

// 自定义 Handler（入站处理器）
public class BusinessHandler extends SimpleChannelInboundHandler {
    @Override
    protected void channelRead0(ChannelHandlerContext ctx, FullHttpRequest msg) {
        String uri = msg.uri();
        System.out.println("处理请求：" + uri);
        // ctx.fireChannelRead(msg); // 继续传递给下一个 Handler
        // 不调用则责任链在此终止
    }
}

面试高频考点

Q1：Filter 和 Interceptor 的区别？各自基于什么模式？
维度Filter（过滤器）Interceptor（拦截器）
规范Servlet 规范，容器级别Spring MVC 特有
拦截范围所有请求（包括静态资源）只拦截 DispatcherServlet 处理的请求
执行时机Servlet 容器层，DispatcherServlet 之前DispatcherServlet 内部，Controller 前后
能获取 Spring Bean需要特殊处理（DelegatingFilterProxy）天然支持 @Autowired
设计模式责任链（FilterChain.doFilter 传递）责任链 + 模板方法（preHandle/postHandle）
Q2：责任链中请求没被任何处理者处理怎么办？
这是责任链的一个潜在问题。常见解决方案：
  • 末尾兜底处理者：在链尾加一个"DefaultHandler"，记录日志或返回默认响应
  • 返回值标记：handle() 返回 boolean，链尾返回 false 表示未处理，调用方判断
  • 抛出异常：链尾的 passToNext() 在 next == null 时抛出异常，强制调用方处理
Spring Security 的方案：ExceptionTranslationFilter 作为兜底，将未处理的认证/授权异常转换为 HTTP 响应。
Q3：如何动态调整责任链的顺序？
Java
// 方案1：List + 排序（最灵活）
List handlers = new ArrayList<>();
handlers.add(new AuthHandler());
handlers.add(new LoggingHandler());
handlers.add(new BusinessHandler());

// 按优先级排序（Handler 实现 Ordered 接口）
handlers.sort(Comparator.comparingInt(Handler::getOrder));

// Spring Security 的 Filter 就是这种方式，每个 Filter 有 order 值
// @Order(1) class LoggingFilter 比 @Order(2) class AuthFilter 先执行

// 方案2：Spring Bean + @Order 注解
@Order(1)
@Component
public class LoggingFilter implements Filter { ... }

@Order(2)
@Component
public class AuthFilter implements Filter { ... }
Q4：责任链 vs 装饰器模式的区别？
两者结构相似（都有链式引用），但意图不同：
  • 责任链：请求沿链传递，某一个处理者处理（纯）或全员都处理（非纯）；关注"谁来处理"
  • 装饰器：所有装饰器都会参与处理，且最终委托到被装饰对象；关注"如何增强"
  • 接口约束：装饰器与组件实现相同接口；责任链的 Handler 之间不要求接口一致
⚠️

实战陷阱

陷阱1：链过长导致性能问题
责任链每一层都有方法调用开销。如果链有几十个处理者，且每个请求都要全部遍历，性能会显著下降。解决：合并功能相近的处理者；对高频路径做短路优化（满足条件立即返回）。
陷阱2：循环链导致死循环
Java
// ❌ 错误：形成循环链，无限递归
HandlerA a = new HandlerA();
HandlerB b = new HandlerB();
a.setNext(b);
b.setNext(a); // 循环！a→b→a→b→...，栈溢出

// ✅ 防御：setNext 时检查是否会形成环
public Approver setNext(Approver next) {
    if (next == this) throw new IllegalArgumentException("不能将自身设为下一个处理者");
    // 更严格：遍历链检查是否已存在 next
    this.next = next;
    return next;
}
陷阱3：Filter 中忘记调用 chain.doFilter() 导致请求被静默丢弃
非纯责任链（Servlet Filter）中，如果某个 Filter 忘记调用 chain.doFilter() 且没有写响应，请求会被静默丢弃，客户端永远等待。解决：明确处理两种情况：短路时写响应，继续时调用 chain.doFilter()。
⚖️

优缺点分析

✅ 优点

  • 解耦发送者与接收者：客户端不需要知道谁处理请求
  • 单一职责：每个处理者只关注自己的逻辑
  • 开闭原则：新增处理者无需修改现有代码
  • 灵活组装：运行时动态调整链的顺序和成员

❌ 缺点

  • 不保证处理：纯责任链中请求可能到达链尾仍未处理
  • 调试困难：请求在多个处理者间流转，难以追踪
  • 性能开销：链过长时每次请求都经历多次方法调用
  • 循环风险：配置不当可能形成循环链

与其他模式的关系

相关模式关系说明
装饰器模式结构相似（链式引用），但责任链关注"谁处理请求"；装饰器关注"如何增强功能"
命令模式可以将命令对象作为请求在责任链中传递，处理者解析命令并执行
组合模式责任链的处理者可以是组合对象（叶子节点或组合节点），形成树形责任结构
观察者模式观察者通知所有订阅者（广播）；责任链是有序的单播（每个请求找到唯一处理者）
