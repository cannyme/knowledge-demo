BIO示例（阻塞IO）

BIO采用一连接一线程模型，每个连接需要独立的线程处理，连接数多时资源消耗大。

Java
// BIO服务端（一连接一线程）
class BIOServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(8080);
        System.out.println("BIO服务器启动...");

        while (true) {
            // 阻塞等待连接
            Socket socket = serverSocket.accept();
            System.out.println("客户端连接：" + socket.getInetAddress());

            // 为每个连接创建新线程
            new Thread(() -> {
                try {
                    BufferedReader reader = new BufferedReader(
                        new InputStreamReader(socket.getInputStream()));
                    PrintWriter writer = new PrintWriter(socket.getOutputStream());

                    String line;
                    while ((line = reader.readLine()) != null) {
                        System.out.println("收到：" + line);
                        writer.println("Echo: " + line);
                        writer.flush();
                    }
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}
⚠️ 缺点：每个连接一个线程，线程切换开销大，连接数受限（几千个就接近极限）。

Buffer工作原理

Java
// Buffer核心属性
// 0 <= mark <= position <= limit <= capacity

public static void bufferDemo() {
    // 创建Buffer
    ByteBuffer buffer = ByteBuffer.allocate(1024);

    // 写入数据（写模式）
    buffer.put("Hello".getBytes());
    // position=5, limit=1024, capacity=1024

    // 切换为读模式（flip）
    buffer.flip();
    // position=0, limit=5, capacity=1024

    // 读取数据
    byte[] data = new byte[buffer.remaining()]; // remaining = limit - position
    buffer.get(data);
    System.out.println(new String(data)); // Hello

    // 清空，准备再次写入（clear）
    buffer.clear();
    // position=0, limit=1024, capacity=1024
}

NIO服务端示例

Java
class NIOServer {
    public static void main(String[] args) throws IOException {
        // 创建Selector
        Selector selector = Selector.open();

        // 创建ServerSocketChannel
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.bind(new InetSocketAddress(8080));
        serverChannel.configureBlocking(false); // 非阻塞模式

        // 注册到Selector，监听连接事件
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);
        System.out.println("NIO服务器启动...");

        while (true) {
            // 阻塞等待事件（可设置超时）
            selector.select();

            // 获取就绪的事件
            Set<SelectionKey> selectedKeys = selector.selectedKeys();
            Iterator<SelectionKey> iterator = selectedKeys.iterator();

            while (iterator.hasNext()) {
                SelectionKey key = iterator.next();
                iterator.remove();

                if (key.isAcceptable()) {
                    // 接受连接
                    ServerSocketChannel server = (ServerSocketChannel) key.channel();
                    SocketChannel client = server.accept();
                    client.configureBlocking(false);
                    client.register(selector, SelectionKey.OP_READ);
                    System.out.println("客户端连接：" + client.getRemoteAddress());

                } else if (key.isReadable()) {
                    // 读取数据
                    SocketChannel client = (SocketChannel) key.channel();
                    ByteBuffer buffer = ByteBuffer.allocate(1024);
                    int bytesRead = client.read(buffer);

                    if (bytesRead == -1) {
                        client.close();
                    } else {
                        buffer.flip();
                        client.write(buffer); // Echo
                    }
                }
            }
        }
    }
}

Reactor模式

Reactor是基于事件驱动的设计模式，是NIO编程的核心架构模式。

单Reactor单线程

flowchart TD R[Reactor] -->|监听| S[Selector] S -->|分发| H1[Handler 1] S -->|分发| H2[Handler 2] S -->|分发| H3[Handler 3] H1 --> P[业务处理] H2 --> P H3 --> P

缺点：一个线程处理所有请求，无法利用多核CPU，handler阻塞会影响整个服务。

单Reactor多线程（Netty采用）

flowchart TD subgraph MainReactor["Boss线程组"] MR[MainReactor] -->|监听连接| SS[ServerSocketChannel] end subgraph SubReactor["Worker线程池"] SR[SubReactor 1] --> H1[Handler] SR2[SubReactor 2] --> H2[Handler] SR3[SubReactor N] --> H3[Handler] end SS -.->|分发| SR SS -.->|分发| SR2 SS -.->|分发| SR3

优点：Boss线程只负责连接建立，Worker线程池负责I/O读写，充分利用多核。

Netty架构设计

Netty是基于NIO的高性能网络框架，封装了NIO的复杂性，提供易用的API。

核心组件

组件说明
EventLoopGroup事件循环组（Boss/Worker）
EventLoop事件循环，处理Channel的I/O操作
Channel网络通道，封装了Socket
ChannelPipeline处理器链，责任链模式
ChannelHandler处理器，处理业务逻辑
ChannelHandlerContext处理器上下文
ByteBuf字节缓冲区，优于NIO ByteBuffer

Netty服务端示例

Java
public class NettyServer {
    public static void main(String[] args) throws Exception {
        // Boss线程组：处理连接（1个即可）
        EventLoopGroup bossGroup = new NioEventLoopGroup(1);
        // Worker线程组：处理I/O（默认CPU核心数*2）
        EventLoopGroup workerGroup = new NioEventLoopGroup();

        try {
            ServerBootstrap bootstrap = new ServerBootstrap();
            bootstrap.group(bossGroup, workerGroup)
                .channel(NioServerSocketChannel.class)
                .option(ChannelOption.SO_BACKLOG, 128)
                .childOption(ChannelOption.SO_KEEPALIVE, true)
                .childHandler(new ChannelInitializer<SocketChannel>() {
                    @Override
                    protected void initChannel(SocketChannel ch) {
                        ch.pipeline()
                            .addLast(new StringDecoder())   // 解码
                            .addLast(new StringEncoder())   // 编码
                            .addLast(new ServerHandler());  // 业务处理器
                    }
                });

            ChannelFuture future = bootstrap.bind(8080).sync();
            System.out.println("Netty服务器启动...");
            future.channel().closeFuture().sync();

        } finally {
            bossGroup.shutdownGracefully();
            workerGroup.shutdownGracefully();
        }
    }
}

@Sharable // 可共享的无状态Handler
public class ServerHandler extends ChannelInboundHandlerAdapter {
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) {
        System.out.println("收到消息：" + msg);
        ctx.writeAndFlush("Echo: " + msg);
    }

    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) {
        cause.printStackTrace();
        ctx.close();
    }
}

// 常用Handler类型
public class HandlerExample {

    // 入站处理器
    public class InboundHandler extends ChannelInboundHandlerAdapter {
        @Override
        public void channelRead(ChannelHandlerContext ctx, Object msg) {
            // 处理读取的数据
            ctx.fireChannelRead(msg); // 传递给下一个Handler
        }

        @Override
        public void channelActive(ChannelHandlerContext ctx) {
            // 连接建立时触发
            ctx.fireChannelActive();
        }
    }

    // 出站处理器
    public class OutboundHandler extends ChannelOutboundHandlerAdapter {
        @Override
        public void write(ChannelHandlerContext ctx, Object msg, ChannelPromise promise) {
            // 处理写出的数据
            ctx.write(msg, promise); // 传递给下一个Handler
        }
    }

    // 编码器
    public class MyEncoder extends MessageToByteEncoder<MyMessage> {
        @Override
        protected void encode(ChannelHandlerContext ctx, MyMessage msg, ByteBuf out) {
            // 将对象编码为字节
            out.writeInt(msg.getLength());
            out.writeBytes(msg.getData());
        }
    }

    // 解码器（处理粘包/拆包）
    public class MyDecoder extends ByteToMessageDecoder {
        @Override
        protected void decode(ChannelHandlerContext ctx, ByteBuf in, List<Object> out) {
            if (in.readableBytes() < 4) return;
            in.markReaderIndex();
            int length = in.readInt();
            if (in.readableBytes() < length) {
                in.resetReaderIndex();
                return;
            }
            byte[] data = new byte[length];
            in.readBytes(data);
            out.add(new MyMessage(length, data));
        }
    }
}

ByteBuf（字节缓冲区）

ByteBuf是Netty的增强版字节缓冲区，相比NIO的ByteBuffer更强大、更易用。

特性ByteBufferByteBuf
读写指针需要flip切换独立的readerIndex/writerIndex
容量扩展固定大小自动扩展
内存类型堆内存堆/堆外内存
引用计数无有（内存池管理）
池化无有（减少GC压力）
Java
// ByteBuf使用示例
public void byteBufDemo() {
    // 1. 创建ByteBuf
    ByteBuf buf = Unpooled.buffer(1024); // 未池化堆内存
    ByteBuf directBuf = Unpooled.directBuffer(1024); // 未池化堆外内存
    // ByteBuf pooledBuf = PooledByteBufAllocator.DEFAULT.buffer(1024); // 池化

    // 2. 写入数据
    buf.writeInt(100);
    buf.writeBytes("Hello".getBytes());

    // 3. 读取数据
    int num = buf.readInt();           // 100
    byte[] data = new byte[5];
    buf.readBytes(data);               // "Hello"

    // 4. 引用计数（池化ByteBuf需要）
    buf.retain();  // 引用计数+1
    buf.release(); // 引用计数-1，为0时释放

    // 5. 零拷贝：切片（共享底层数据）

    // 6. 零拷贝：组合（CompositeByteBuf）
    CompositeByteBuf composite = Unpooled.compositeBuffer();
    composite.addComponents(true, buf1, buf2, buf3);
}

Java
// Netty零拷贝实现
public class ZeroCopyExample {

    // 1. FileRegion：使用sendfile系统调用传输文件
    public void sendFile(ChannelHandlerContext ctx, File file) throws IOException {
        FileRegion region = new DefaultFileRegion(
            new FileInputStream(file).getChannel(), 0, file.length());
        ctx.writeAndFlush(region);
    }

    // 2. CompositeByteBuf：逻辑合并多个Buffer，无需物理复制
    public void compositeBuffer() {
        ByteBuf header = Unpooled.wrappedBuffer(new byte[]{0x01, 0x02});
        ByteBuf body = Unpooled.wrappedBuffer(new byte[]{0x03, 0x04});

        CompositeByteBuf composite = Unpooled.compositeBuffer();
        composite.addComponents(true, header, body);
        // 逻辑上是一个Buffer，物理上共享数据
    }

    // 3. slice：切片，共享底层数据
    public void sliceBuffer() {
        ByteBuf buf = Unpooled.wrappedBuffer("Hello World".getBytes());
        ByteBuf slice1 = buf.slice(0, 5);   // "Hello"
        ByteBuf slice2 = buf.slice(6, 5);   // "World"
        // 共享buf的底层数组，没有数据拷贝
    }

    // 4. wrappedBuffer：包装现有数组，避免拷贝
    public void wrapBuffer() {
        byte[] data = new byte[1024];
        ByteBuf buf = Unpooled.wrappedBuffer(data);
        // 直接使用data数组，不拷贝
    }
}

粘包/拆包与心跳机制

粘包/拆包问题

  • 粘包：多个小消息被合并成一个大数据包发送
  • 拆包：一个大数据包被拆分成多个小包发送
  • 原因：TCP是流协议，不保证消息边界

解决方案

Java
// 1. 固定长度解码器
ch.pipeline().addLast(new FixedLengthFrameDecoder(1024));

// 2. 行分隔符解码器
ch.pipeline().addLast(new LineBasedFrameDecoder(8192));

// 3. 分隔符解码器
ch.pipeline().addLast(new DelimiterBasedFrameDecoder(
    8192, Unpooled.wrappedBuffer(new byte[]{'$'})));

// 4. 长度字段解码器（推荐）
// 协议格式：| 长度(4字节) | 内容(N字节) |
ch.pipeline().addLast(new LengthFieldBasedFrameDecoder(
    65535,   // 最大帧长度
    0,       // 长度字段偏移
    4,       // 长度字段长度
    0,       // 长度调整
    4        // 跳过的字节数（长度字段本身）
));

// 心跳处理器
public class HeartbeatHandler extends ChannelDuplexHandler {
    @Override
    public void userEventTriggered(ChannelHandlerContext ctx, Object evt) {
        if (evt instanceof IdleStateEvent) {
            IdleStateEvent event = (IdleStateEvent) evt;
            if (event.state() == IdleState.READER_IDLE) {
                // 读超时，关闭连接
                ctx.close();
            } else if (event.state() == IdleState.WRITER_IDLE) {
                // 写超时，发送心跳
                ctx.writeAndFlush(new PingMessage());
            }
        }
    }
}

Netty最佳实践

方面建议
线程数配置Boss线程1个；Worker线程CPU核心数*2
内存配置使用池化DirectBuffer；合理设置ByteBuf大小
Handler设计避免在Handler中执行耗时操作；使用@Sharable注解共享无状态Handler
资源释放使用ReferenceCountUtil.release()；或使用SimpleChannelInboundHandler自动释放
日志使用LoggingHandler调试，生产环境移除或降低级别
💡 业务线程池：耗时业务操作应提交到业务线程池处理，避免阻塞EventLoop线程
Java
// 业务线程池处理耗时操作
public class BusinessHandler extends ChannelInboundHandlerAdapter {
    private final ExecutorService executor = Executors.newFixedThreadPool(100);

    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) {
        // 提交到业务线程池
        executor.submit(() -> {
            try {
                // 执行业务逻辑
                Object result = process(msg);
                // 通过ctx写回响应（保证线程安全）
                ctx.writeAndFlush(result);
            } catch (Exception e) {
                e.printStackTrace();
            }
        });
    }
}
