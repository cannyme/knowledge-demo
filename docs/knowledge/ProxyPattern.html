代理模式

为其他对象提供一种代理以控制对这个对象的访问。代理对象在客户端和目标对象之间起到中介作用，可以实现访问控制、延迟加载、日志记录等功能。

🎯

模式定义与意图

代理模式（Proxy Pattern）是一种结构型设计模式，它为其他对象提供一种代理以控制对这个对象的访问。代理对象可以在客户端和目标对象之间起到中介的作用，并附加额外的功能。

核心思想：通过引入代理对象来控制对目标对象的访问，在不修改目标对象的前提下，增加额外的功能或控制逻辑。

生活中的类比：

  • 房产中介：代替房东处理租房事务
  • VPN代理：代替用户访问被限制的网络资源
  • 律师代理：代替委托人处理法律事务
  • 游戏加速器：代理游戏连接，降低延迟
🏗️

架构原理

classDiagram class Subject { <> +request() } class RealSubject { +request() } class Proxy { -realSubject: RealSubject +request() } class Client Subject <|.. RealSubject Subject <|.. Proxy Proxy --> RealSubject Client --> Subject

角色说明：

角色说明
Subject抽象主题，定义代理和真实主题的公共接口
RealSubject真实主题，定义代理所代表的真实对象
Proxy代理主题，持有对真实主题的引用，可以控制对真实主题的访问
Client客户端，通过Subject接口与代理交互

代理类型实现

静态代理

代理类在编译时已经确定，需要为每个被代理类编写对应的代理类。

Java
// 接口
interface UserService {
    void addUser(String name);
    void deleteUser(String name);
}

// 真实主题
class UserServiceImpl implements UserService {
    @Override
    public void addUser(String name) {
        System.out.println("添加用户：" + name);
    }

    @Override
    public void deleteUser(String name) {
        System.out.println("删除用户：" + name);
    }
}

// 静态代理
class UserServiceProxy implements UserService {
    private final UserService target;

    public UserServiceProxy(UserService target) {
        this.target = target;
    }

    @Override
    public void addUser(String name) {
        System.out.println("[静态代理] 记录日志 - 添加用户前");
        long start = System.currentTimeMillis();
        target.addUser(name);
        System.out.println("[静态代理] 记录日志 - 添加用户后，耗时："
            + (System.currentTimeMillis() - start) + "ms");
    }

    @Override
    public void deleteUser(String name) {
        System.out.println("[静态代理] 记录日志 - 删除用户前");
        target.deleteUser(name);
        System.out.println("[静态代理] 记录日志 - 删除用户后");
    }
}

// 使用
UserService proxy = new UserServiceProxy(new UserServiceImpl());
proxy.addUser("张三");

JDK动态代理

基于接口实现，运行时生成代理类。使用java.lang.reflect.Proxy和InvocationHandler。

Java
import java.lang.reflect.*;

// 调用处理器
class JdkProxyHandler implements InvocationHandler {
    private final Object target;

    public JdkProxyHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args)
            throws Throwable {
        System.out.println("[JDK代理] 方法调用前：" + method.getName());
        long start = System.currentTimeMillis();
        try {
            // 调用目标方法
            Object result = method.invoke(target, args);
            System.out.println("[JDK代理] 方法调用成功");
            return result;
        } catch (Exception e) {
            System.out.println("[JDK代理] 方法调用异常：" + e.getMessage());
            throw e;
        } finally {
            System.out.println("[JDK代理] 耗时："
                + (System.currentTimeMillis() - start) + "ms");
        }
    }

    // 创建代理对象的工厂方法
    @SuppressWarnings("unchecked")
    public static <T> T createProxy(T target) {
        return (T) Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            new JdkProxyHandler(target)
        );
    }
}

// 使用
UserService target = new UserServiceImpl();
UserService proxy = JdkProxyHandler.createProxy(target);
proxy.addUser("李四");  // 自动触发invoke方法
JDK动态代理原理：
  • 运行时生成字节码：ProxyGenerator.generateProxyClass()
  • 生成的代理类继承Proxy，实现目标接口
  • 代理类持有InvocationHandler引用
  • 调用代理方法时，实际调用handler.invoke()

CGLIB代理

基于继承实现，通过生成目标类的子类来创建代理。可以代理没有实现接口的类。

Java
import net.sf.cglib.proxy.*;

// CGLIB方法拦截器
class CglibProxyInterceptor implements MethodInterceptor {

    @Override
    public Object intercept(Object obj, Method method,
                           Object[] args, MethodProxy proxy) throws Throwable {
        System.out.println("[CGLIB代理] 方法调用前：" + method.getName());
        long start = System.currentTimeMillis();
        try {
            // 调用父类（目标类）的方法
            Object result = proxy.invokeSuper(obj, args);
            System.out.println("[CGLIB代理] 方法调用成功");
            return result;
        } catch (Exception e) {
            System.out.println("[CGLIB代理] 方法调用异常：" + e.getMessage());
            throw e;
        } finally {
            System.out.println("[CGLIB代理] 耗时："
                + (System.currentTimeMillis() - start) + "ms");
        }
    }

    // 创建代理对象
    @SuppressWarnings("unchecked")
    public static <T> T createProxy(Class<T> targetClass) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(targetClass);
        enhancer.setCallback(new CglibProxyInterceptor());
        return (T) enhancer.create();
    }
}

// 目标类（无需实现接口）
class OrderService {
    public void createOrder(String orderId) {
        System.out.println("创建订单：" + orderId);
    }
}

// 使用
OrderService proxy = CglibProxyInterceptor.createProxy(OrderService.class);
proxy.createOrder("ORD-001");
💡 CGLIB特点
  • 基于ASM字节码技术，生成目标类的子类
  • 无法代理final类和方法
  • 性能比JDK动态代理稍高（无需反射调用）
  • Spring AOP默认使用CGLIB代理无接口的类

保护代理示例

Java
// 文档接口
interface Document {
    void read();
    void edit(String content);
    void remove();
}

// 真实文档
class RealDocument implements Document {
    private final String name;
    public RealDocument(String name) { this.name = name; }

    @Override public void read() {
        System.out.println("读取文档：" + name);
    }
    @Override public void edit(String content) {
        System.out.println("编辑文档：" + name + "，内容：" + content);
    }
    @Override public void remove() {
        System.out.println("删除文档：" + name);
    }
}

// 保护代理
class DocumentProxy implements Document {
    private final RealDocument document;
    private final String role;

    public DocumentProxy(RealDocument doc, String role) {
        this.document = doc;
        this.role = role;
    }

    @Override public void read() {
        document.read();  // 所有人可读
    }

    @Override public void edit(String content) {
        if (!"admin".equals(role) && !"editor".equals(role)) {
            System.out.println("权限不足：无法编辑文档");
            return;
        }
        document.edit(content);
    }

    @Override public void remove() {
        if (!"admin".equals(role)) {
            System.out.println("权限不足：无法删除文档");
            return;
        }
        document.remove();
    }
}

// 使用
RealDocument doc = new RealDocument("机密文档.txt");
Document guestDoc = new DocumentProxy(doc, "guest");
guestDoc.read();        // 成功
guestDoc.edit("test");  // 权限不足
guestDoc.remove();      // 权限不足

Spring AOP中的代理选择

Spring AOP是代理模式的经典应用，它自动选择代理方式：

Java
// Spring AOP配置
@Configuration
@EnableAspectJAutoProxy(proxyTargetClass = false)  // JDK动态代理
@EnableAspectJAutoProxy(proxyTargetClass = true)   // CGLIB代理
public class AopConfig {
}

// 切面定义
@Aspect
@Component
public class LoggingAspect {
    @Before("execution(* com.example.service.*.*(..))")
    public void logBefore(JoinPoint joinPoint) {
        System.out.println("方法调用前：" + joinPoint.getSignature().getName());
    }

    @Around("execution(* com.example.service.*.*(..))")
    public Object logAround(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.currentTimeMillis();
        Object result = pjp.proceed();  // 执行目标方法
        long duration = System.currentTimeMillis() - start;
        System.out.println("方法执行时间：" + duration + "ms");
        return result;
    }
}
条件Spring选择说明
目标类实现了接口JDK动态代理默认行为，proxyTargetClass=false
目标类未实现接口CGLIB代理强制使用CGLIB
proxyTargetClass=trueCGLIB代理强制使用CGLIB
目标类是final无法代理CGLIB无法继承final类

代理 vs 装饰器

代理模式和装饰器模式在结构上非常相似，但意图不同：

维度代理模式装饰器模式
目的控制访问增强功能
关注点对象访问的控制和中介动态添加职责
关系代理和被代理者是关联关系装饰者和被装饰者是聚合关系
透明度对客户端透明，无法感知代理存在可以层层包装，显式增强
生命周期代理通常控制对象生命周期装饰者不负责对象创建
