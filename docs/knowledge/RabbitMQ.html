RabbitMQ 消息队列

RabbitMQ 是基于 AMQP 协议的开源消息队列，采用 Erlang 编写，以灵活的路由能力和高可靠性著称，适合复杂的企业级业务场景。

📖

概念说明类比
Producer消息生产者，创建并发送消息寄件人
Exchange交换器，接收消息并按规则路由到队列邮局分拣中心
Queue消息队列，存储消息的缓冲区邮箱
Consumer消息消费者，从队列获取并处理消息收件人
Binding绑定，Exchange 与 Queue 的路由规则收件地址
Routing Key路由键，Exchange 路由消息的依据邮编

Exchange 交换器类型

Direct Exchange - 精确匹配

完全匹配路由键，点对点发送。

graph LR P[Producer] -->|routingKey=order| E[Direct Exchange] E -->|bindingKey=order| Q1[Order Queue] E -.->|bindingKey=user| Q2[User Queue] style E fill:#3a2e15,stroke:#f0b429,color:#f0b429 style Q1 fill:#1a3a1a,stroke:#7ee787,color:#7ee787
// 声明 Direct Exchange
channel.exchangeDeclare("order.exchange", "direct");
channel.queueBind("order.queue", "order.exchange", "order.create");
Topic Exchange - 模式匹配

支持通配符匹配：* 匹配一个单词，# 匹配零个或多个单词。

graph LR P[Producer] -->|order.create.success| E[Topic Exchange] E -->|order.*.*| Q1[All Order Events] E -->|order.create.#| Q2[Create Events] E -->|order.*.success| Q3[Success Events] style E fill:#3a2e15,stroke:#f0b429,color:#f0b429
// 声明 Topic Exchange
channel.exchangeDeclare("order.topic", "topic");
channel.queueBind("order.all.queue", "order.topic", "order.*.*");
channel.queueBind("order.create.queue", "order.topic", "order.create.#");
Fanout Exchange - 广播

忽略路由键，广播到所有绑定的队列。

graph LR P[Producer] --> E[Fanout Exchange] E --> Q1[SMS Notification] E --> Q2[Email Notification] E --> Q3[Push Notification] style E fill:#3a2e15,stroke:#f0b429,color:#f0b429
// 声明 Fanout Exchange（广播）
channel.exchangeDeclare("notification.fanout", "fanout");
channel.queueBind("sms.queue", "notification.fanout", "");
channel.queueBind("email.queue", "notification.fanout", "");
Headers Exchange - 头属性匹配

根据消息头属性进行匹配，支持 x-match: any 或 all。

// Headers Exchange 匹配示例
Map<String, Object> headers = new HashMap<>();
headers.put("x-match", "all");  // 需要匹配所有 headers
headers.put("type", "vip");
headers.put("level", "gold");

channel.queueBind("vip.queue", "headers.exchange", "", headers);
类型路由规则适用场景
Direct精确匹配 routing key点对点消息、简单路由
Topic模式匹配 * 和 #日志分级、事件订阅
Fanout广播到所有队列通知推送、多副本处理
Headers匹配 header 属性复杂属性筛选

Java 使用示例

生产者

Java
@Component
public class RabbitProducer {
    @Autowired
    private RabbitTemplate rabbitTemplate;

    // Direct 发送
    public void sendDirect(String routingKey, Object message) {
        rabbitTemplate.convertAndSend("order.exchange", routingKey, message);
    }

    // Topic 发送
    public void sendTopic(String routingKey, Object message) {
        rabbitTemplate.convertAndSend("order.topic", routingKey, message);
    }

    // Fanout 发送（无需 routing key）
    public void sendFanout(Object message) {
        rabbitTemplate.convertAndSend("notification.fanout", "", message);
    }

    // 发送带确认
    public void sendWithConfirm(String routingKey, Object message) {
        rabbitTemplate.setConfirmCallback((correlationData, ack, cause) -> {
            if (ack) {
                System.out.println("消息已送达 Exchange");
            } else {
                System.err.println("消息投递失败: " + cause);
            }
        });
        rabbitTemplate.convertAndSend("order.exchange", routingKey, message);
    }
}

消费者

Java
@Component
public class RabbitConsumer {

    // Direct 消费
    @RabbitListener(queues = "order.queue")
    public void handleOrder(Order order) {
        System.out.println("收到订单: " + order.getOrderId());
        processOrder(order);
    }

    // Topic 消费 - 监听所有订单创建事件
    @RabbitListener(bindings = @QueueBinding(
        value = @Queue("order.create.queue"),
        exchange = @Exchange(value = "order.topic", type = ExchangeTypes.TOPIC),
        key = "order.create.#"
    ))
    public void handleOrderCreate(Order order) {
        System.out.println("订单创建: " + order.getOrderId());
    }

    // 手动 ACK 模式
    @RabbitListener(queues = "critical.queue", ackMode = "MANUAL")
    public void handleCritical(Order order, Channel channel,
                               @Header(AmqpHeaders.DELIVERY_TAG) long tag) throws IOException {
        try {
            processCriticalOrder(order);
            // 手动确认
            channel.basicAck(tag, false);
        } catch (Exception e) {
            // 拒绝并重新入队
            channel.basicNack(tag, false, true);
        }
    }
}

延迟队列

Java
// 方式一：TTL + 死信队列实现延迟
@Bean
public Queue delayQueue() {
    Map<String, Object> args = new HashMap<>();
    args.put("x-message-ttl", 30000);           // 30秒过期
    args.put("x-dead-letter-exchange", "normal.exchange");
    args.put("x-dead-letter-routing-key", "order.routing.key");
    return QueueBuilder.durable("delay.queue").withArguments(args).build();
}

// 方式二：使用延迟队列插件（推荐）
@Bean
public CustomExchange delayedExchange() {
    Map<String, Object> args = new HashMap<>();
    args.put("x-delayed-type", "direct");
    return new CustomExchange("delayed.exchange", "x-delayed-message", true, false, args);
}

// 发送延迟消息
rabbitTemplate.convertAndSend("delayed.exchange", "order.routing.key", order, msg -> {
    msg.getMessageProperties().setDelay(30000);  // 延迟 30 秒
    return msg;
});

RabbitMQ 特点总结

特性RabbitMQ
核心优势灵活路由（多种 Exchange 类型）、AMQP 标准协议
单机吞吐量万级（5-10万）
消息延迟微秒级
路由能力极强（Direct/Topic/Fanout/Headers）
延迟队列支持（TTL + DLX 或插件）
消息回溯不支持
适用场景复杂路由、企业集成、RPC 调用、任务分发
💡 选型建议：
RabbitMQ 适合需要复杂路由规则、灵活消息分发的场景。如果业务需要精准的消息路由（如按用户等级、订单类型等），RabbitMQ 是最佳选择。但对于超大规模数据流处理，Kafka 更合适。
