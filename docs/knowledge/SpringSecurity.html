Spring Security核心概念

Spring Security是一个功能强大且高度可定制的安全框架，为Java应用提供认证、授权和防护功能。本文详细解析其核心组件、认证流程、JWT实现、OAuth2.0以及最佳实践。

🎯

核心功能

Spring Security提供三大核心安全功能：

功能说明典型应用场景
认证（Authentication）验证用户身份，确认"你是谁"用户登录验证、JWT令牌校验
授权（Authorization）验证用户权限，确认"你能做什么"角色权限控制、方法级安全
防护（Protection）防御常见Web攻击CSRF防护、CORS配置、Session固定攻击防护
💡
认证 vs 授权的区别 认证解决的是"证明你是你"的问题，通常通过用户名密码、指纹、Token等方式验证身份。授权解决的是"你能做什么"的问题，在认证通过后决定用户可以访问哪些资源。
🏗️

核心组件

Spring Security的核心组件构成了完整的认证授权体系：

组件职责说明
SecurityContext安全上下文容器存储当前线程的Authentication对象，通过SecurityContextHolder访问
Authentication认证信息封装包含principal（用户信息）、credentials（凭证）、authorities（权限列表）
UserDetails用户详情接口定义用户核心属性：用户名、密码、权限、账户状态等
UserDetailsService用户加载服务根据用户名加载UserDetails，是连接数据库的桥梁
AuthenticationProvider认证提供者执行实际认证逻辑，如DaoAuthenticationProvider验证用户名密码
FilterChain过滤器链Spring Security基于Filter实现，多个安全过滤器串联处理请求
🔄

认证流程详解

Spring Security的认证流程是一个多阶段的协作过程：

sequenceDiagram actor User as 用户 participant Filter as UsernamePassword
AuthenticationFilter participant AuthMgr as AuthenticationManager participant Provider as DaoAuthenticationProvider participant UDS as UserDetailsService participant Encoder as PasswordEncoder participant Context as SecurityContext User->>Filter: 提交用户名密码 Filter->>Filter: 创建UsernamePasswordAuthenticationToken Filter->>AuthMgr: authenticate(token) AuthMgr->>Provider: 委托认证 Provider->>UDS: loadUserByUsername(username) UDS-->>Provider: 返回UserDetails Provider->>Encoder: matches(rawPwd, encodedPwd) Encoder-->>Provider: 验证结果 Provider-->>AuthMgr: 返回认证成功的Authentication AuthMgr-->>Filter: 返回认证结果 Filter->>Context: SecurityContextHolder.setContext() Filter-->>User: 认证成功，进入系统
💡
流程要点
  • UsernamePasswordAuthenticationToken是未认证时的载体，包含用户提交的凭据
  • ProviderManager是AuthenticationManager的默认实现，管理多个Provider
  • 认证成功后，Token会被标记为已认证，并存入SecurityContext

基础配置示例

Spring Security 6.x推荐使用SecurityFilterChain进行配置：

java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            // ========== 授权配置 ==========
            .authorizeHttpRequests(auth -> auth
                // 公开路径
                .requestMatchers("/public/**", "/login", "/register").permitAll()
                // 管理员权限
                .requestMatchers("/admin/**").hasRole("ADMIN")
                // 用户权限（支持多个角色）
                .requestMatchers("/user/**").hasAnyRole("USER", "ADMIN")
                // 特定权限
                .requestMatchers("/api/orders").hasAuthority("ORDER_READ")
                // 其他请求需认证
                .anyRequest().authenticated()
            )

            // ========== 表单登录 ==========
            .formLogin(form -> form
                .loginPage("/login")              // 自定义登录页
                .loginProcessingUrl("/doLogin")   // 登录处理URL
                .defaultSuccessUrl("/home")       // 登录成功跳转
                .failureUrl("/login?error")       // 登录失败跳转
                .usernameParameter("username")    // 用户名参数名
                .passwordParameter("password")    // 密码参数名
                .permitAll()
            )

            // ========== 注销配置 ==========
            .logout(logout -> logout
                .logoutUrl("/logout")
                .logoutSuccessUrl("/login?logout")
                .invalidateHttpSession(true)      // 使Session失效
                .deleteCookies("JSESSIONID")      // 删除Cookie
                .permitAll()
            )

            // ========== CSRF配置 ==========
            .csrf(csrf -> csrf
                .ignoringRequestMatchers("/api/**") // API路径禁用CSRF
                // .disable()  // 完全禁用（不推荐）
            )

            // ========== Session管理 ==========
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED)
                .maximumSessions(1)               // 单点登录
                .maxSessionsPreventsLogin(false)  // 后登录踢掉先登录
            )

            // ========== 记住我 ==========
            .rememberMe(remember -> remember
                .key("uniqueAndSecretKey")
                .tokenValiditySeconds(86400)      // 1天
            );

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public AuthenticationManager authenticationManager(
            AuthenticationConfiguration config) throws Exception {
        return config.getAuthenticationManager();
    }
}

UserDetailsService实现

UserDetailsService是连接Spring Security与数据库的桥梁，负责根据用户名加载用户信息：

java
@Service
public class CustomUserDetailsService implements UserDetailsService {

    @Autowired
    private UserRepository userRepository;

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        // 1. 查询数据库
        User user = userRepository.findByUsername(username)
            .orElseThrow(() -> new UsernameNotFoundException("用户不存在: " + username));

        // 2. 构建权限列表
        List<SimpleGrantedAuthority> authorities = user.getRoles().stream()
            .flatMap(role -> {
                // 添加角色权限
                Stream<String> perms = Stream.of("ROLE_" + role.getName());
                // 添加具体权限
                Stream<String> rolePerms = role.getPermissions().stream()
                    .map(Permission::getName);
                return Stream.concat(perms, rolePerms);
            })
            .map(SimpleGrantedAuthority::new)
            .collect(Collectors.toList());

        // 3. 返回UserDetails对象
        return new org.springframework.security.core.userdetails.User(
            user.getUsername(),
            user.getPassword(),
            user.isEnabled(),                    // 账户是否可用
            user.isAccountNonExpired(),          // 账户是否过期
            user.isCredentialsNonExpired(),      // 凭证是否过期
            user.isAccountNonLocked(),           // 账户是否锁定
            authorities
        );
    }
}
⚠️
实现注意事项
  • 用户不存在时应抛出UsernameNotFoundException，不要返回null
  • 角色名需要添加"ROLE_"前缀，权限名不需要
  • 建议将账户状态（锁定、过期等）也持久化到数据库

JWT认证实现

JWT（JSON Web Token）是现代前后端分离应用的首选认证方案，具有无状态、跨域、易扩展等优势。

1. JWT工具类

java
@Component
public class JwtUtils {

    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expiration:86400000}")
    private Long expiration;  // 默认1天

    // 生成签名密钥
    private SecretKey getSigningKey() {
        byte[] keyBytes = Decoders.BASE64.decode(secret);
        return Keys.hmacShaKeyFor(keyBytes);
    }

    /**
     * 生成JWT令牌
     */
    public String generateToken(UserDetails userDetails) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("roles", userDetails.getAuthorities().stream()
            .map(GrantedAuthority::getAuthority)
            .collect(Collectors.toList()));

        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + expiration);

        return Jwts.builder()
            .claims(claims)
            .subject(userDetails.getUsername())
            .issuedAt(now)
            .expiration(expiryDate)
            .signWith(getSigningKey(), Jwts.SIG.HS256)
            .compact();
    }

    /**
     * 从Token中提取用户名
     */
    public String extractUsername(String token) {
        Claims claims = Jwts.parser()
            .verifyWith(getSigningKey())
            .build()
            .parseSignedClaims(token)
            .getPayload();
        return claims.getSubject();
    }

    /**
     * 验证Token是否有效
     */
    public boolean validateToken(String token, UserDetails userDetails) {
        try {
            String username = extractUsername(token);
            return username.equals(userDetails.getUsername()) && !isTokenExpired(token);
        } catch (JwtException | IllegalArgumentException e) {
            return false;
        }
    }

    /**
     * 检查Token是否过期
     */
    private boolean isTokenExpired(String token) {
        Date expiration = Jwts.parser()
            .verifyWith(getSigningKey())
            .build()
            .parseSignedClaims(token)
            .getPayload()
            .getExpiration();
        return expiration.before(new Date());
    }
}

2. JWT过滤器

java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired
    private JwtUtils jwtUtils;

    @Autowired
    private UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain)
                                    throws ServletException, IOException {
        try {
            // 1. 从请求中提取Token
            String token = extractToken(request);

            // 2. 验证Token并设置认证
            if (token != null && jwtUtils.validateToken(token)) {
                String username = jwtUtils.extractUsername(token);
                UserDetails userDetails = userDetailsService.loadUserByUsername(username);

                // 3. 创建认证对象
                UsernamePasswordAuthenticationToken authentication =
                    new UsernamePasswordAuthenticationToken(
                        userDetails,
                        null,
                        userDetails.getAuthorities()
                    );

                // 4. 设置请求详情
                authentication.setDetails(
                    new WebAuthenticationDetailsSource().buildDetails(request)
                );

                // 5. 存入SecurityContext
                SecurityContextHolder.getContext().setAuthentication(authentication);
            }
        } catch (Exception e) {
            logger.error("无法设置用户认证: {}", e.getMessage());
        }

        filterChain.doFilter(request, response);
    }

    /**
     * 从Authorization头提取Bearer Token
     */
    private String extractToken(HttpServletRequest request) {
        String header = request.getHeader("Authorization");
        if (StringUtils.hasText(header) && header.startsWith("Bearer ")) {
            return header.substring(7);
        }
        return null;
    }
}

3. 注册过滤器

java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http,
                                       JwtAuthenticationFilter jwtFilter) throws Exception {
    http
        // 禁用Session（JWT无状态）
        .sessionManagement(session ->
            session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        // 禁用CSRF（使用JWT代替）

        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/**").permitAll()
            .anyRequest().authenticated()
        );
    return http.build();
}

常用注解

注解作用时机示例
@PreAuthorize方法执行前@PreAuthorize("hasRole('ADMIN')")
@PostAuthorize方法执行后@PostAuthorize("returnObject.owner == authentication.name")
@PostFilter方法返回后@PostFilter("filterObject.userId == authentication.name")
@PreFilter方法执行前过滤参数@PreFilter("filterObject.status == 'ACTIVE'")
@SecuredJSR-250风格@Secured("ROLE_ADMIN")

使用示例

java
@Service
public class OrderService {

    @Autowired
    private OrderRepository orderRepository;

    // 只有ADMIN可以删除订单
    @PreAuthorize("hasRole('ADMIN')")
    public void deleteOrder(Long orderId) {
        orderRepository.deleteById(orderId);
    }

    // 只有订单所有者或ADMIN可以查看
    @PreAuthorize("@orderService.canAccessOrder(#orderId, authentication.name) or hasRole('ADMIN')")
    public Order getOrder(Long orderId) {
        return orderRepository.findById(orderId)
            .orElseThrow(() -> new OrderNotFoundException(orderId));
    }

    // 方法执行后验证：确保返回的订单属于当前用户
    @PostAuthorize("returnObject.userId == authentication.name")
    public Order getOwnOrder(Long orderId) {
        return orderRepository.findById(orderId).orElse(null);
    }

    // 过滤返回列表，只保留当前用户的数据
    @PostFilter("filterObject.userId == authentication.name")
    public List<Order> getMyOrders() {
        return orderRepository.findAll();
    }

    // 自定义权限评估方法
    public boolean canAccessOrder(Long orderId, String username) {
        Order order = orderRepository.findById(orderId).orElse(null);
        return order != null && order.getUserId().equals(username);
    }
}
💡
SpEL表达式支持 注解中可以使用Spring Expression Language (SpEL)：
  • authentication - 当前认证对象
  • #paramName - 方法参数（需要@Param或编译带参数名）
  • returnObject - 返回值（仅@PostAuthorize）
  • filterObject - 集合中的元素（仅@PreFilter/@PostFilter）

OAuth2.0授权流程

OAuth2.0是业界标准的授权协议，允许第三方应用获取有限的资源访问权限，而无需暴露用户凭据。

授权码模式流程

sequenceDiagram actor User as 用户 participant Client as 客户端应用 participant Auth as 授权服务器
(Authorization Server) participant Resource as 资源服务器
(Resource Server) User->>Client: 1. 点击登录按钮 Client->>Auth: 2. 重定向到授权服务器 Auth->>User: 3. 展示登录/授权页面 User->>Auth: 4. 登录并同意授权 Auth->>Client: 5. 重定向回调（携带授权码code） Client->>Auth: 6. 用code + client_secret换取Token Auth->>Client: 7. 返回access_token + refresh_token Client->>Resource: 8. 携带access_token访问API Resource->>Resource: 9. 验证Token有效性 Resource->>Client: 10. 返回受保护资源

授权模式对比

模式适用场景安全性推荐度
授权码模式Web应用、移动端高⭐⭐⭐ 推荐
简化模式（Implicit）单页应用（已废弃）中不推荐
密码凭证可信的第一方应用低谨慎使用
客户端凭证服务间调用中适用
设备码智能电视等输入受限设备中适用
PKCE移动端/SPA（授权码增强）高⭐⭐⭐ 推荐

资源服务器配置

java
@Configuration
public class OAuth2ResourceServerConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer(oauth2 -> oauth2
                // JWT Token配置
                .jwt(jwt -> jwt
                    .jwtAuthenticationConverter(jwtAuthenticationConverter())
                    .decoder(jwtDecoder())
                )
                // 或Opaque Token配置（远程校验）
                // .opaqueToken(opaque -> opaque
                //     .introspectionUri("/oauth2/introspect")
                //     .introspectionClientCredentials("client", "secret")
                // )
            );
        return http.build();
    }

    @Bean
    public JwtAuthenticationConverter jwtAuthenticationConverter() {
        JwtAuthenticationConverter converter = new JwtAuthenticationConverter();
        // 配置权限映射
        converter.setJwtGrantedAuthoritiesConverter(jwt -> {
            List<String> roles = jwt.getClaimAsStringList("roles");
            return roles.stream()
                .map(role -> new SimpleGrantedAuthority("ROLE_" + role))
                .collect(Collectors.toList());
        });
        // 配置principal映射
        converter.setPrincipalClaimName("sub");
        return converter;
    }

    @Bean
    public JwtDecoder jwtDecoder() {
        return ReactiveJwtDecoders.fromIssuerLocation("https://auth-server.com");
    }
}

CSRF防护原理

CSRF（Cross-Site Request Forgery，跨站请求伪造）是一种常见的Web攻击方式，Spring Security默认提供防护。

攻击原理

sequenceDiagram actor User as 用户 participant Bank as 银行网站A participant Evil as 恶意网站B User->>Bank: 1. 登录银行网站，获取Cookie Bank-->>User: Set-Cookie: sessionId=xxx Note over User: 用户保持登录状态 User->>Evil: 2. 访问恶意网站B Evil->>Evil: 3. 页面包含隐藏表单
<form action="bank.com/transfer"> Evil->>Bank: 4. 自动提交表单（携带用户Cookie） Bank->>Bank: 5. 验证通过（有合法Cookie） Bank-->>User: 6. 转账完成（用户不知情）

防护方案

方案原理Spring Security支持
CSRF Token服务端生成随机Token，校验请求时携带默认开启，支持Cookie/Session存储
SameSite Cookie限制Cookie在跨站请求中发送需浏览器+服务器配合
Referer检查验证请求来源域名可自定义过滤器实现
自定义Header要求请求携带特定Header适合AJAX请求

前后端分离的CSRF处理

java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        // API路径使用JWT，禁用CSRF
        .csrf(csrf -> csrf
            .ignoringRequestMatchers("/api/**")
        )
        // 或：完全禁用（不推荐用于传统Web应用）
        // .csrf(csrf -> csrf.disable())

        // SPA应用配置：将CSRF Token存入Cookie
        .csrf(csrf -> csrf
            .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
            .csrfTokenRequestHandler(new CsrfTokenRequestAttributeHandler())
        );
    return http.build();
}
⚠️
重要提醒
  • 传统表单提交应用：必须开启CSRF防护
  • 前后端分离+JWT应用：可以禁用CSRF，因为JWT本身不依赖Cookie
  • 混合应用：对API路径禁用，对表单路径保持开启

常见问题Q&A

Q1: Session和JWT应该怎么选择？

选择Session：传统Web应用、服务端渲染、需要服务端控制会话（如强制下线）、小规模应用。

选择JWT：前后端分离、移动端API、微服务架构、需要水平扩展、无状态要求高的场景。

Q2: 如何实现"记住我"功能？

Spring Security提供Remember-Me功能，基于持久化Token或Cookie实现：

java
.rememberMe(remember -> remember
    .key("uniqueAndSecretKey")  // 加密密钥
    .tokenValiditySeconds(86400) // Token有效期（秒）
    .tokenRepository(persistentTokenRepository()) // 持久化存储
    .userDetailsService(userDetailsService)
)
Q3: 密码加密应该选择什么算法？

强烈推荐使用BCrypt，它是目前最安全的哈希算法之一：

  • 自动加盐，每个密码的盐值不同
  • 可配置工作因子（strength），默认10，越高越安全但越慢
  • Spring Security的BCryptPasswordEncoder已内置

避免使用：MD5、SHA-1（已被彩虹表破解）；避免自定义加密方案。

Q4: 如何动态加载权限？

默认权限在启动时加载，动态权限需要自定义FilterInvocationSecurityMetadataSource：

java
@Component
public class DynamicSecurityMetadataSource implements FilterInvocationSecurityMetadataSource {

    @Autowired
    private PermissionService permissionService;

    @Override
    public Collection<ConfigAttribute> getAttributes(Object object) {
        // 从数据库实时查询URL对应的权限
        FilterInvocation fi = (FilterInvocation) object;
        String url = fi.getRequestUrl();
        return permissionService.getAttributesByUrl(url);
    }
}
Q5: 如何处理跨域（CORS）？

@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .cors(cors -> cors.configurationSource(corsConfigurationSource()))

@Bean
public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowedOrigins(Arrays.asList("http://localhost:3000"));
    config.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE"));
    config.setAllowedHeaders(Arrays.asList("*"));
    config.setAllowCredentials(true);

    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config);
    return source;
}

JWT + Spring Security 实战配置

JWT（JSON Web Token）与 Spring Security 结合是现代无状态认证的主流方案：

java
// SecurityConfig - Spring Security 6.x 配置风格
@Configuration
@EnableWebSecurity
@EnableMethodSecurity   // 启用方法级权限控制 @PreAuthorize
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(AbstractHttpConfigurer::disable)          // 无状态API禁用CSRF
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/auth/**", "/public/**").permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint(jwtAuthEntryPoint)  // 未认证401
                .accessDeniedHandler(jwtAccessDeniedHandler)  // 无权限403
            )
            .build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();  // 密码加密器
    }
}

// JWT 过滤器
@Component
public class JwtAuthFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest req,
                                    HttpServletResponse res,
                                    FilterChain chain) throws IOException, ServletException {
        String token = extractToken(req);
        if (token != null && jwtUtil.validateToken(token)) {
            UsernamePasswordAuthenticationToken auth =
                new UsernamePasswordAuthenticationToken(
                    jwtUtil.getUsername(token), null,
                    jwtUtil.getAuthorities(token)
                );
            SecurityContextHolder.getContext().setAuthentication(auth);
        }
        chain.doFilter(req, res);
    }
}

面试高频考点

Q1: Spring Security 的过滤器链是如何工作的？
Spring Security 基于 Servlet Filter 链实现，核心是 DelegatingFilterProxy 将请求委托给 FilterChainProxy，后者管理多条 SecurityFilterChain。常见过滤器执行顺序：SecurityContextPersistenceFilter → UsernamePasswordAuthenticationFilter → BasicAuthenticationFilter → ExceptionTranslationFilter → FilterSecurityInterceptor。每个过滤器处理特定职责，可通过配置增删过滤器。
Q2: Authentication 和 Authorization 的区别？Spring Security 如何实现？
Authentication（认证）：验证你是谁（用户名密码、JWT、OAuth2）；Authorization（授权）：验证你能做什么（角色权限）。Spring Security 通过 AuthenticationManager 完成认证，将结果存入 SecurityContextHolder；通过 AccessDecisionManager（或 Spring 6.x 的 AuthorizationManager）完成授权，支持 @PreAuthorize、@Secured、URL 规则等方式。
Q3: Session 和 JWT 的优缺点对比？
Session：服务端存储，状态管理方便，但多节点部署需共享 Session（Redis），扩展性差。JWT：无状态，自包含用户信息，天然支持分布式，但 Token 无法主动失效（需维护黑名单），payload 过大增加网络开销。实际项目中可结合使用：JWT 携带基本信息，敏感权限变更时加 Redis 黑名单短期记录。
Q4: @PreAuthorize 和 @Secured 的区别？
@Secured 是 Spring Security 原生注解，只能做简单的角色判断（如 @Secured("ROLE_ADMIN")）；@PreAuthorize 支持 SpEL 表达式，功能更强大（如 @PreAuthorize("hasRole('ADMIN') and #id == authentication.principal.id")），可做细粒度的方法级权限控制。需开启 @EnableMethodSecurity 才能使用。
⚡

实战陷阱

⚠️
陷阱1：CSRF 与前后端分离接口冲突 Spring Security 默认开启 CSRF 保护（适合服务端渲染场景）。前后端分离 + JWT 无状态方案必须禁用 CSRF（http.csrf(AbstractHttpConfigurer::disable)），否则所有 POST/PUT/DELETE 请求会返回 403 Forbidden，且错误信息不明显。
⚠️
陷阱2：@PreAuthorize 在同类方法调用时失效 与 @Transactional 一样，@PreAuthorize 基于 AOP 代理实现，同类内部 this.method() 调用会绕过权限检查。确保安全方法从外部调用，或使用 @EnableMethodSecurity(proxyTargetClass=true)。
⚠️
陷阱3：密码明文存储 不使用 PasswordEncoder 导致密码明文存储是严重安全漏洞。必须使用 BCryptPasswordEncoder（推荐），其强度参数 strength 默认为 10（2^10 次哈希），可根据服务器性能调整（12-14 更安全）。绝不可在生产环境使用 NoOpPasswordEncoder。
